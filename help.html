分形树桌面壁纸 使用帮助

简介

分形树桌面壁纸能够随机生成一棵漂亮的树，得益于其强大的参数系统，这几乎可以模拟世界上任何类型的树，包括自然界存在的和只存在于你想象中的

什么是分形树？

分形树是一种递归生成的树结构————每个树枝都有多个子树枝，每个子树枝又包含多个子树枝，以此类推，即可得到一个树状结构

在分形树桌面壁纸中，为了减少渲染时的计算量和可控性，我们只使用二叉树，即每个树枝最多有两个子树枝

为了实现尽可能真实的树形结构，分形树桌面壁纸提供了大量参数，用于控制节点间的距离、角度，连线颜色等

参数入门

如果你刚接触 分形树桌面壁纸，不必被大量参数吓到————你只需要调整几个核心参数，就能创造出一棵独一无二的树。试着修改分叉角度范围和长度衰减范围，看看树木形态如何变化；再调整叶片设置中的颜色，为你的树增添色彩。当你熟悉基础操作后，可以进一步探索参数组和分布曲线，让每一棵树都成为一件真正的艺术品

以下提供一些基础的参数教学↓

1. 分叉角度范围

分叉角度范围控制了子树枝相对于父树枝的偏转角度范围（度）。通过分布曲线控制角度值在范围内的分布概率（当然，分布曲线不在基础教学范围内，保持默认均匀分布即可）

  • 当分叉角度范围总体较大时，树冠会显得稀疏蓬松
  • 当分叉角度范围总体较小时，树冠会显得密密麻麻
  • 当分叉角度范围区间较广时，树冠会显得杂乱无章
  • 当分叉角度范围区间较窄时，树冠会显得整齐有序

2. 长度衰减范围

长度衰减范围控制了子树枝长度相对于父树枝长度的比例范围。通过分布曲线控制比例值在范围内的分布概率（当然，分布曲线一样不在基础教学范围内，保持默认均匀分布即可）

例如一次分叉，长度衰减在预设范围中随机到0.7，则子树枝长度为父树枝长度的70%

  • 当长度衰减范围总体较大时，树冠会显得巨大无比
  • 当长度衰减范围总体较小时，树冠会显得矮小稀疏
  • 当长度衰减范围区间较广时，树冠会显得错落有致
  • 当长度衰减范围区间较窄时，树冠会显得鳞次齿比

注意，如果没有特殊需求，长度衰减范围的最大值最好不要超过0.95，否则会导致树冠过密，甚至无限增长（只能靠最大深度硬限制），影响渲染性能不说，树冠看起来也会极其不自然

3. 叶片设置

叶片设置控制了叶片的形态。通过控制叶片颜色、权重、密度、线宽，实现不同的叶片效果。（对于基础教学，只修改颜色即可）

可以先试着同时添加几个有一定差别的绿色，看看效果如何

4. 背景颜色

顾名思义，背景颜色控制了画布的背景颜色

可以直接使用浅浅的树叶色调，更加自然

想寻求更加专业的色彩建议，可以参考颜色搭配生成器–色轮工具 | Adobe Express

参数进阶

本部分将对各参数给出详细的定义、用法。

所有以像素为单位的参数，均表示窗口高度为标准1080像素时的长度，实际渲染时依据窗口高度等比缩放。

1. 全局设置

全局设置不包含于任何参数组中，直接对效果产生影响。

1.1 树名

树名是用来给你的树起名字的。

用于主界面列表显示，并作为导出图片的默认文件名前缀。空字符串会被自动修正为"未命名"。

1.2 背景颜色

背景颜色是用来控制画布背景颜色的。

用于窗口背景色及壁纸图片背景色的填充。

2. 基本形态参数

以下参数直接控制树的基本形态、终止条件。

2.1 主干长度

主干长度是从树根发出的第一级树枝的初始长度（像素）。树根定位于画布底部上方 10% 处。

用于决定整棵树的基准尺度大小，直接影响树的最终大小。

2.2 最短分支长度

最短分支长度是树的终止条件之一。当树枝长度低于此阈值时，不再产生子分枝。

用于控制树冠末端的细密程度和树冠的大小。

2.3 最大深度

最大深度是树的终止条件之一。当树枝的递归层数（树根为第 0 层，树根分出的子树枝为第 1 层，以此类推）达到此值后停止分叉。

用于防止树冠过密或无限增长，影响性能，甚至直接卡死。

注意，创作前期最大深度建议设置为10 ~ 30，后续根据需要调整，以防测试时参数不合理导致的卡死。

2.4 叶子阈值

叶子阈值是用来决定当前分支应该作为枝条还是叶子的阈值。根据分支长度划分视觉角色。当前分支长度低于此阈值时，该枝条按叶片渲染；高于阈值时按树干渲染。

用于控制叶子的视觉效果，影响树冠的形态。

通常不建议用来直接调整叶子密度，因为这和叶子的形态相关，应通过调整其他参数来实现。

2.5 枝条最大倾角

树枝最大倾角是树的终止条件之一。控制子枝绝对角度（相对于竖直向上方向的夹角）的上限（度）。任何子枝角度超出此值将不被创建，直接终止。

用于防止树枝下垂，保持树的形态。

3. 分叉角度范围

分叉角度范围是每个子分支各自独立抽样的偏转角度值区间范围，两个子分支分别向父分支两侧偏转。抽样范围为 [最小值, 最大值]（度）。

用于控制分支的形态，影响分支的整齐程度和生长方式。

3.1 角度概率分布曲线

通过分布曲线按钮打开条形图编辑器：点击柱状图调整柱子高度，高度范围为 0.02 ~ 1.0。

每个子分支的抽样方式：将角度区间等分为 8 个子区间，先以分布曲线的柱状图高度为权重选中一个子区间，再在该子区间内均匀随机取值。若未指定分布曲线，则在全局区间均匀随机。

简单理解就是柱子越高，附近的值被选中作为分支依据的概率越高。

3.2 参考示例

默认树种松树的分叉角度范围为 [0, 90]，角度概率分布曲线大致如下：

实际效果是松树大多笔直生长，有时却大幅突转，能够提供类似现实中松树虬曲遒劲的形态。

其中，更可能平行于父分支，为了保持树冠的基本形态不至于过于混乱。

4. 长度衰减范围

长度衰减范围是每个子分支各自独立抽样的长度衰减比例区间范围，子分支实际长度 = 父枝长度 × 抽样得到的衰减比例。抽样范围为 [最小值, 最大值]（衰减比例）。

4.1 衰减概率分布曲线

与3.1 角度概率分布曲线类似。

通过分布曲线按钮打开条形图编辑器：点击柱状图调整柱子高度，高度范围为 0.02 ~ 1.0。

每个子分支的抽样方式：将角度区间等分为 8 个子区间，先以分布曲线的柱状图高度为权重选中一个子区间，再在该子区间内均匀随机取值。若未指定分布曲线，则在全局区间均匀随机。

简单理解就是柱子越高，附近的值被选中作为衰减比例的概率越高。

4.2 参考示例

默认树种绿树的长度衰减范围为 [0.6, 0.94]，衰减概率分布曲线大致如下：

可见绿树的衰减比例更可能取到 0.60 和 0.94 左右。也就是说，绿树的分支更要么几乎与父分支长度相同，要么明显更短。

实际效果是绿树有时枝条舒展，有时则收束长叶，能够提供绿树层次分明、错落有致的形态。

值得注意的是，如果绿树过早选择短分支，会直接减少整棵树的高度，进而使树高差别较大。

5. 树干/叶片设置

树干/叶片设置是用于决定树干与叶片视觉效果的参数集合，不会影响树的形态结构。

树干与叶片各有独立的颜色列表。每条颜色条目含三个属性：

5.1 颜色

颜色是用于决定树干或叶片渲染用颜色的参数。支持给树干/叶片设置多种颜色，随机选取。

5.2 权重

权重是用于决定该颜色在绘制新树干或叶片时被选中的可能性的参数。值越大，该颜色在树干/叶片中的占比越高。

5.3 线宽比

线宽比是用于决定树干或叶片宽度系数的参数。值越大，树干/叶片越粗，实际粗细还取决于分支长度。

多参数组行为：树干/叶片设置不参与多参数组的数值加权。当存在多个参数组时，选择权重最大的参数组的树干/叶片设置直接使用。

6. 落叶系统

落叶系统是用于在地面增加落叶的功能。当一片叶片被绘制后，有概率在地面区域额外绘制一片形态相同的落叶。

用于提升树的氛围感。

6.1 落叶密度

落叶密度是用于决定叶片转化为地面落叶的概率的参数。取值范围为0.0 ~ 1.0。值越大，落叶数量越多。

具体的落叶数量还取决于树冠的密度（树上的叶片数量）。

6.2 落叶分散程度

落叶分散程度是用于决定落叶分布范围的参数，具体指落叶相对于原叶片的水平偏移范围（像素）和相对于地面位置的竖直偏移范围（像素）。值越大，落叶分布范围越大。

相对于地面位置的竖直偏移采用三角分布（两次均匀随机取平均）。

相对于原叶片的水平偏移采用均匀分布，因为大量树叶的随机已经能够起到类似正态分布的效果。

7. 参数组系统

参数组是分形树桌面壁纸的核心抽象概念。单一树种配置可包含一至多个参数组，各组拥有一套独立的参数。

7.0 为什么需要参数组

设想一下，你希望设计一棵树干笔直，树冠却舒展的树。

如果只使用一组参数，那么树干与树冠的形态只能相同，无法实现复杂的效果。

事实上，这正是我设计枫树时真实遇到的问题。

为此，我有过两种设想

第一种设想，是将分支y坐标纳入“参数的参数”,能够直接实现参数随高度的变化，换句话说，参数可以写成关于y的函数。

这很符合程序员思维，并且能够实现更加复杂精确的形态控制。但很明显，这极其不直观。实际操作时，需要相当复杂的抽象思维，且借助计算器，甚至是Desmos等工具来研究函数图像，反复调参。

因此我选择了第二种设想————参数组系统。

通过直接定义多套参数，在不同的高度应用不同的参数组，来区分树干与树冠的形态。

实际开发时，为了过渡自然，我选择了对不同参数组的参数进行反比加权混合后应用。

7.1 基本操作

在多参数组控制中：

  • 下拉框： 选择参数组以进行编辑，点击可修改参数组名称。
  • “+”： 添加一个新的参数组（复制当前选中参数组的参数）。
  • “✕”： 删除当前选中的参数组。

7.2 典型生效高度

典型生效高度是该参数组的锚点高度（像素），用于多参数组的加权混合计算。

用于控制参数组生效的大致位置，分支距离地面的高度越接近典型生效高度，参数组的权重越高。

简单理解就是参数会在典型生效高度附近生效。

7.3 权重计算

当枝条位于高度 $h$ 时，第 $i$ 组的原始权重为：

w_i = \min\left(\frac{1}{|h - t_i|},\ 128\right)

其中 $t_i$ 为第 $i$ 组的典型高度。距离为 0 时权重固定为 128，避免除零。所有权重归一化后：

  • 普通标量参数（角度范围、衰减范围、最短分支、最大深度、叶子阈值、落叶参数、主干长度、枝条倾角）：加权平均。
  • 树干/叶片设置：取最大权重组的颜色列表直接使用。
  • 分布曲线（角度分布、衰减分布）：逐柱子加权平均。

仅一组参数组时跳过混合计算，直接使用该组全部参数。

7.4 参考示例

默认树种枫树拥有“主干”与“树冠”两个参数组。二者仅有的区别是分叉角度范围、长度衰减范围、典型生效高度。

“主干” 参数组：

  • 典型生效高度为 0，越接近地面，参数组权重越高。
  • 分叉角度范围为 0 ~ 10 均匀分布，维持树干直立的形态。
  • 长度衰减范围为 0.8 ~ 0.95 均匀分布，保证树干较长，减少下方分支。

“树冠” 参数组：

  • 典型生效高度为 180，基本上越远离地面，参数组权重越高。
  • 分叉角度范围为 0 ~ 60 两端概率高，维持树冠蓬松的形态。
  • 长度衰减范围为 0.3 ~ 0.95 大端概率高，小端概率低但仍有可能，维持树冠错落的形态，增加树冠的多样性。

导入/导出树种

程序会将所有自定义树种保存至AppData\Local\分形树桌面壁纸\trees文件夹中的<树名>.json文件中，可以点击参数设置窗口左下角打开树文件夹按钮快捷打开。

  • 如需导入树种，直接将需要导入的<树名>.json文件放入该文件夹中，后重启程序即可。
  • 如需导出树种，直接将需要导出的<树名>.json文件复制即可。

渲染窗口

主界面点击任意树种按钮或编辑界面点击预览后打开分形树渲染窗口，支持拖拽缩放。

  • 刷新：重置随机种子并重绘，每次点击形态各异。
  • 保存为图片：以窗口大小渲染为 PNG，保存至桌面<树名>_<时间戳>.png，完成后不关闭窗口。
  • 设为桌面壁纸：以屏幕分辨率 2 倍渲染为 BMP，调用系统接口设为 Windows 桌面壁纸，完成后窗口自动关闭。

Fractal Tree Desktop Wallpaper - User Guide

Note: This document is AI-translated from the original Chinese version. If you find any errors or unclear expressions, please refer to the original help.md file.

Introduction

Fractal Tree Desktop Wallpaper can randomly generate a beautiful tree. Thanks to its powerful parameter system, it can simulate almost any type of tree in the world, including those that exist in nature and those that only exist in your imagination.

What is a Fractal Tree?

A Fractal Tree is a recursively generated tree structure — each branch has multiple sub-branches, each sub-branch contains multiple sub-branches, and so on, resulting in a tree-like structure.

In Fractal Tree Desktop Wallpaper, to reduce rendering computation and improve controllability, we only use binary trees, meaning each branch has at most two sub-branches.

To achieve a tree structure that is as realistic as possible, Fractal Tree Desktop Wallpaper provides numerous parameters to control the distance and angle between nodes, line colors, and more.

Getting Started with Parameters

If you're new to Fractal Tree Desktop Wallpaper, don't be intimidated by the many parameters — you only need to adjust a few core parameters to create a unique tree. Try modifying the Branch Angle Range and Length Decay Range to see how the tree's shape changes; then adjust the color in Leaf Settings to add color to your tree. Once you're familiar with the basic operations, you can further explore Parameter Groups and Distribution Curves to make each tree a true work of art.

Here's some basic parameter tutorials below ↓

1. Branch Angle Range

Branch Angle Range controls the range of deflection angles (in degrees) of sub-branches relative to their parent branch. The distribution curve controls the probability of angle values within the range (of course, distribution curves are beyond the scope of this basic tutorial, so just keep the default uniform distribution).

  • When Branch Angle Range is generally large, the tree crown appears sparse and fluffy.
  • When Branch Angle Range is generally small, the tree crown appears dense and compact.
  • When Branch Angle Range has a wide interval, the tree crown appears chaotic and disorderly.
  • When Branch Angle Range has a narrow interval, the tree crown appears neat and orderly.

2. Length Decay Range

Length Decay Range controls the range of the ratio of sub-branch length to parent branch length. The distribution curve controls the probability of ratio values within the range (again, distribution curves are beyond the scope of this basic tutorial, so just keep the default uniform distribution).

For example, if a branch gets a random value of 0.7 from the preset range, the sub-branch length will be 70% of the parent branch length.

  • When Length Decay Range is generally large, the tree crown appears enormous.
  • When Length Decay Range is generally small, the tree crown appears short and sparse.
  • When Length Decay Range has a wide interval, the tree crown appears varied and layered.
  • When Length Decay Range has a narrow interval, the tree crown appears uniform and dense.

Note that unless you have special requirements, the maximum value of Length Decay Range should not exceed 0.95, otherwise the tree crown will be too dense, or even grow infinitely (limited only by Max Depth), which not only affects rendering performance but also makes the tree crown look extremely unnatural.

3. Leaf Settings

Leaf Settings controls the appearance of leaves. By adjusting leaf color, weight, density, and line width, you can achieve different leaf effects. (For basic tutorials, just modify the colors.)

You can start by adding several shades of green with some variation to see the effect.

4. Background Color

As the name suggests, Background Color controls the canvas background color.

You can use a light leaf-like tone for a more natural look.

For more professional color suggestions, you can refer to Color Wheel Tool – Adobe Express.

Advanced Parameters

This section provides detailed definitions and usage of each parameter.

All parameters in pixels represent lengths when the window height is standard 1080 pixels. Actual rendering scales proportionally based on window height.

1. Global Settings

Global settings are not included in any parameter group and directly affect the result.

1.1 Tree Name

Tree Name is used to name your tree.

It's used for display in the main interface list and as the default file name prefix for exported images. Empty strings are automatically corrected to "Untitled".

1.2 Background Color

Background Color controls the canvas background color.

Used to fill the window background and wallpaper image background.

2. Basic Shape Parameters

The following parameters directly control the basic shape and termination conditions of the tree.

2.1 Trunk Length

Trunk Length is the initial length (in pixels) of the first-level branch emanating from the tree root. The tree root is positioned at 10% above the bottom of the canvas.

Used to determine the baseline scale of the entire tree, directly affecting the tree's final size.

2.2 Minimum Branch Length

Minimum Branch Length is one of the tree's termination conditions. When a branch length falls below this threshold, no more sub-branches are generated.

Used to control the density of the tree crown's ends and the size of the crown.

2.3 Max Depth

Max Depth is one of the tree's termination conditions. When the recursion level of a branch (the root is level 0, sub-branches from the root are level 1, and so on) reaches this value, branching stops.

Used to prevent the tree crown from being too dense or growing infinitely, which would affect performance or even freeze the application.

Note that in the early stages of creation, it's recommended to set Max Depth to 10 ~ 30, and adjust later as needed to prevent freezing during testing due to unreasonable parameters.

2.4 Leaf Threshold

Leaf Threshold determines whether the current branch should be rendered as a twig or a leaf. Based on branch length to determine visual role. When the current branch length is below this threshold, the branch is rendered as a leaf; above the threshold, it's rendered as a trunk.

Used to control the visual effect of leaves, affecting the shape of the tree crown.

Generally, it's not recommended to use this to directly adjust leaf density, as this is related to leaf shape. Other parameters should be adjusted instead.

2.5 Maximum Branch Inclination

Maximum Branch Inclination is one of the tree's termination conditions. Controls the upper limit (in degrees) of the absolute angle of sub-branches (the angle relative to the vertical upward direction). Any sub-branch angle exceeding this value will not be created and will terminate directly.

Used to prevent branches from drooping and maintain the tree's shape.

3. Branch Angle Range

Branch Angle Range is the deflection angle interval range independently sampled for each sub-branch. Two sub-branches deflect to either side of the parent branch respectively. The sampling range is [Min, Max] (degrees).

Used to control branch shape, affecting branch neatness and growth pattern.

3.1 Angle Probability Distribution Curve

Open the bar chart editor through the Distribution Curve button: click the bars to adjust bar heights, with a height range of 0.02 ~ 1.0.

Sampling method for each sub-branch: Divide the angle interval equally into 8 sub-intervals, first select a sub-interval using the bar chart heights of the distribution curve as weights, then take a uniformly random value within that sub-interval. If no distribution curve is specified, a uniformly random value is taken from the global interval.

Simply put, the taller the bar, the higher the probability that nearby values will be selected as branch angles.

5. Trunk/Leaf Settings

Trunk/Leaf Settings is a collection of parameters that determine the visual effect of trunks and leaves, without affecting the tree's structural shape.

Trunks and leaves each have independent color lists. Each color entry contains three attributes:

5.1 Color

Color determines the color used for rendering trunks or leaves. Supports setting multiple colors for trunks/leaves, randomly selected.

5.2 Weight

Weight determines the probability that this color will be selected when drawing a new trunk or leaf. The larger the value, the higher the proportion of this color in trunks/leaves.

5.3 Line Width Ratio

Line Width Ratio determines the width coefficient of trunks or leaves. The larger the value, the thicker the trunk/leaf. The actual thickness also depends on branch length.

Multi-Parameter Group Behavior: Trunk/leaf settings do not participate in numerical weighting of multiple parameter groups. When multiple parameter groups exist, the trunk/leaf settings of the parameter group with the largest weight are used directly.

6. Fallen Leaf System

Fallen Leaf System is a function to add fallen leaves on the ground. After a leaf is drawn, there is a probability of drawing an additional fallen leaf with the same shape in the ground area.

Used to enhance the atmosphere of the tree.

6.1 Fallen Leaf Density

Fallen Leaf Density determines the probability of leaves being converted to ground fallen leaves. The value range is 0.0 ~ 1.0. The larger the value, the more fallen leaves.

The actual number of fallen leaves also depends on the density of the tree crown (the number of leaves on the tree).

6.2 Fallen Leaf Dispersion

Fallen Leaf Dispersion determines the distribution range of fallen leaves, specifically the horizontal offset range (in pixels) relative to the original leaf and the vertical offset range (in pixels) relative to the ground position. The larger the value, the wider the distribution of fallen leaves.

The vertical offset relative to the ground position uses a triangular distribution (average of two uniform random values).

The horizontal offset relative to the original leaf uses a uniform distribution, because the randomness of a large number of leaves already achieves an effect similar to a normal distribution.

7. Parameter Group System

Parameter Group is the core abstraction concept of Fractal Tree Desktop Wallpaper. A single tree species configuration can contain one or more parameter groups, each with an independent set of parameters.

7.0 Why Parameter Groups are Needed

Imagine you want to design a tree with a straight trunk but a spreading crown.

If you only use one set of parameters, the trunk and crown can only have the same shape, making it impossible to achieve complex effects.

In fact, this was the real problem I encountered when designing Maple.

For this, I had two ideas:

The first idea was to incorporate branch y-coordinate into "parameters of parameters", which could directly implement parameters that change with height. In other words, parameters could be written as functions of y.

This aligns well with programmer thinking and can achieve more complex and precise shape control. However, it's clearly extremely unintuitive. In practice, it requires quite complex abstract thinking, and needs calculators or even tools like Desmos to study function graphs and repeatedly adjust parameters.

Therefore, I chose the second idea — the parameter group system.

By directly defining multiple sets of parameters and applying different parameter groups at different heights, the shapes of trunks and crowns can be distinguished.

During actual development, for smooth transitions, I chose to apply inverse-weighted mixing of parameters from different parameter groups.

7.1 Basic Operations

In Multi-Parameter Group Control:

  • Dropdown: Select a parameter group to edit. Click to modify the parameter group name.
  • "+": Add a new parameter group (copies parameters from the currently selected parameter group).
  • "✕": Delete the currently selected parameter group.

7.2 Typical Effective Height

Typical Effective Height is the anchor height (in pixels) of this parameter group, used for weighted mixing calculations of multiple parameter groups.

Used to control the approximate position where the parameter group takes effect. The closer the branch is to the typical effective height from the ground, the higher the weight of the parameter group.

Simply put, parameters will take effect near the Typical Effective Height.

7.3 Weight Calculation

When a branch is at height $h$, the original weight of group $i$ is:

w_i = \min\left(\frac{1}{|h - t_i|},\ 128\right)

Where $t_i$ is the typical height of group $i$. Weight is fixed at 128 when distance is 0 to avoid division by zero. After normalizing all weights:

  • Ordinary scalar parameters (angle range, decay range, minimum branch, max depth, leaf threshold, fallen leaf parameters, trunk length, branch inclination): Weighted average.
  • Trunk/Leaf Settings: Use the color list of the maximum weight group directly.
  • Distribution Curves (angle distribution, decay distribution): Weighted average per bar.

When there is only one parameter group, skip mixing calculation and use all parameters of that group directly.

7.4 Reference Example

The default tree species Maple has two parameter groups: "Trunk" and "Crown". The only differences between them are Branch Angle Range, Length Decay Range, and Typical Effective Height.

"Trunk" parameter group:

  • Typical effective height is 0, meaning the closer to the ground, the higher the parameter group weight.
  • Branch angle range is 0 ~ 10 uniform distribution, maintaining the upright form of the trunk.
  • Length decay range is 0.8 ~ 0.95 uniform distribution, ensuring the trunk is long and reducing lower branches.

"Crown" parameter group:

  • Typical effective height is 180, basically meaning the further from the ground, the higher the parameter group weight.
  • Branch angle range is 0 ~ 60 with high probability at both ends, maintaining the fluffy form of the crown.
  • Length decay range is 0.3 ~ 0.95 with high probability at the large end and low but still possible probability at the small end, maintaining the layered form of the crown and increasing crown diversity.

はじめに

フラクタルツリー デスクトップ壁紙は、美しい木をランダムに生成することができます。強力なパラメータシステムにより、自然界に存在する木から想像上の木まで、世界中のほぼすべての種類の木をシミュレートすることができます。

フラクタルツリーとは？

フラクタルツリーは再帰的に生成される木構造です。各枝は複数の小枝を持ち、各小枝はさらに複数の小枝を含み、これを繰り返すことで木のような構造が形成されます。

フラクタルツリー デスクトップ壁紙では、レンダリング時の計算量を減らし、制御性を高めるために、各枝が最大で2つの小枝しか持たない二分木のみを使用しています。

可能な限りリアルな木の構造を実現するため、フラクタルツリー デスクトップ壁紙は多数のパラメータを提供しており、ノード間の距離、角度、線の色などを制御できます。

パラメータ入門

フラクタルツリー デスクトップ壁紙を初めて使う方は、多数のパラメータに驚く必要はありません。いくつかの核心的なパラメータを調整するだけで、独自の木を作成できます。分岐角度範囲と長さ減衰範囲を変更して、木の形がどのように変化するか見てみましょう。次に葉の設定の色を調整して、木に彩りを加えましょう。基本的な操作に慣れたら、パラメータグループと分布曲線をさらに探求し、一本一本の木を本物の芸術作品にすることができます。

以下は基本的なパラメータのチュートリアルです ↓

1. 分岐角度範囲

分岐角度範囲は、親枝に対する子枝の偏向角度の範囲（度）を制御します。分布曲線によって、範囲内での角度値の分布確率を制御します（分布曲線は基礎チュートリアルの範囲外ですので、デフォルトの均一分布のままで構いません）。

  • 分岐角度範囲が全体的に大きい場合、樹冠はまばらでふわふわに見えます。
  • 分岐角度範囲が全体的に小さい場合、樹冠は密でぎっしりに見えます。
  • 分岐角度範囲の範囲が広い場合、樹冠は雑然として無秩序に見えます。
  • 分岐角度範囲の範囲が狭い場合、樹冠は整然として秩序正しく見えます。

2. 長さ減衰範囲

長さ減衰範囲は、親枝の長さに対する子枝の長さの比率範囲を制御します。分布曲線によって、範囲内での比率値の分布確率を制御します（これも基礎チュートリアルの範囲外ですので、デフォルトの均一分布のままで構いません）。

例えば、ある分岐で長さ減衰がプリセット範囲から0.7がランダムに選ばれた場合、子枝の長さは親枝の長さの70%になります。

  • 長さ減衰範囲が全体的に大きい場合、樹冠は巨大に見えます。
  • 長さ減衰範囲が全体的に小さい場合、樹冠は低くてまばらに見えます。
  • 長さ減衰範囲の範囲が広い場合、樹冠は変化に富んで層状的に見えます。
  • 長さ減衰範囲の範囲が狭い場合、樹冠は均一で密に見えます。

注意：特別な要件がない限り、長さ減衰範囲の最大値は0.95を超えないことをお勧めします。そうしないと、樹冠が過密になり、無限に成長してしまい（最大深度で制限するしかない）、レンダリングのパフォーマンスに影響するだけでなく、樹冠が極めて不自然に見えます。

3. 葉の設定

葉の設定は葉の形態を制御します。葉の色、重み、密度、線幅を制御して、異なる葉の効果を実現します。（基礎チュートリアルでは、色を変更するだけで十分です。）

まず、いくつかの異なる緑色を同時に追加して、効果を確認してみましょう。

4. 背景色

その名の通り、背景色はキャンバスの背景色を制御します。

薄い葉の色調を使用すると、より自然に見えます。

より専門的な色彩の提案については、カラーホイールツール | Adobe Expressをご参照ください。

パラメータ応用

このセクションでは、各パラメータの詳細な定義と使用方法を説明します。

ピクセル単位のパラメータはすべて、ウィンドウの高さが標準の1080ピクセルの場合の長さを表します。実際のレンダリングでは、ウィンドウの高さに応じて比例してスケーリングされます。

1. グローバル設定

グローバル設定はパラメータグループに含まれず、効果に直接影響します。

1.1 木の名前

木の名前は木に名前を付けるために使用します。

メイン画面のリストに表示され、エクスポートする画像のデフォルトのファイル名のプレフィックスとして使用されます。空の文字列は自動的に「無題」に修正されます。

1.2 背景色

背景色はキャンバスの背景色を制御するために使用します。

ウィンドウの背景色と壁紙画像の背景色の塗りつぶしに使用されます。

2. 基本形状パラメータ

以下のパラメータは、木の基本形状と終了条件を直接制御します。

2.1 幹の長さ

幹の長さは、根から発する最初のレベルの枝の初期長さ（ピクセル）です。根はキャンバスの下から10%上の位置に配置されます。

木全体の基準スケールを決定し、木の最終的なサイズに直接影響します。

2.2 最小枝の長さ

最小枝の長さは、木の終了条件の一つです。枝の長さがこの閾値を下回ると、小枝は生成されなくなります。

樹冠の末端の細かさと樹冠のサイズを制御します。

2.3 最大深度

最大深度は、木の終了条件の一つです。枝の再帰レベル（根はレベル0、根から分かれた子枝はレベル1、以下同様）がこの値に達すると、分岐が停止します。

樹冠が過密になったり無限に成長してパフォーマンスに影響したり、アプリケーションがフリーズするのを防ぎます。

注意：作成初期には、最大深度を10〜30に設定し、パラメータが不適切でテスト中にフリーズするのを防ぐため、後で必要に応じて調整してください。

2.4 葉の閾値

葉の閾値は、現在の枝を小枝としてレンダリングするか葉としてレンダリングするかを決定する閾値です。枝の長さに基づいて視覚的な役割を決定します。現在の枝の長さがこの閾値を下回ると、その枝は葉としてレンダリングされ、閾値を超えると幹としてレンダリングされます。

葉の視覚効果を制御し、樹冠の形状に影響します。

通常、葉の密度を直接調整するために使用することはお勧めしません。これは葉の形状に関連しているため、他のパラメータを調整して実現してください。

2.5 最大枝傾斜角

最大枝傾斜角は、木の終了条件の一つです。子枝の絶対角度（垂直上向き方向に対する角度）の上限（度）を制御します。この値を超える子枝は作成されず、直接終了します。

枝が垂れ下がるのを防ぎ、木の形状を維持します。

3. 分岐角度範囲

分岐角度範囲は、各子枝が独立してサンプリングする偏向角度の区間範囲です。2つの子枝はそれぞれ親枝の両側に偏向します。サンプリング範囲は [最小値, 最大値]（度）です。

枝の形状を制御し、枝の整然性と成長パターンに影響します。

3.1 角度確率分布曲線

分布曲線ボタンで棒グラフエディタを開きます：棒をクリックして棒の高さを調整します。高さの範囲は 0.02 ~ 1.0 です。

各子枝のサンプリング方法: 角度区間を8つの小区間に等分割し、分布曲線の棒グラフの高さを重みとして小区間を選択し、その小区間内で均一にランダムな値を取ります。分布曲線が指定されていない場合、全体区間で均一にランダムな値を取ります。

簡単に言うと、棒が高いほど、その付近の値が枝の角度として選ばれる確率が高くなります。

長さ減衰範囲は、各子枝が独立してサンプリングする長さ減衰比率の区間範囲です。子枝の実際の長さ = 親枝の長さ × サンプリングされた減衰比率。サンプリング範囲は [最小値, 最大値]（減衰比率）です。

0.89
0.94

緑の木の減衰比率が0.60と0.94付近を取りやすいことがわかります。つまり、緑の木の枝は親枝とほぼ同じ長さか、明らかに短いかのどちらかです。

実際の効果は、緑の木は時に枝が広がり、時には葉を集め、層状で変化に富んだ形状を提供します。

なお、緑の木が早い段階で短い枝を選択すると、木全体の高さが直接減少し、木の高さのばらつきが大きくなります。

5. 幹/葉の設定

幹/葉の設定は、幹と葉の視覚効果を決定するパラメータの集合で、木の構造には影響しません。

幹と葉はそれぞれ独立した色リストを持ちます。各色エントリには3つの属性があります：

5.1 色

色は、幹または葉のレンダリングに使用する色を決定するパラメータです。幹/葉に複数の色を設定でき、ランダムに選択されます。

5.2 重み

重みは、新しい幹または葉を描画する際にこの色が選ばれる確率を決定するパラメータです。値が大きいほど、その色の幹/葉での割合が高くなります。

5.3 線幅比

線幅比は、幹または葉の幅係数を決定するパラメータです。値が大きいほど、幹/葉が太くなります。実際の太さは枝の長さにも依存します。

複数パラメータグループの動作: 幹/葉の設定は複数パラメータグループの数値加重平均には参加しません。複数のパラメータグループが存在する場合、重みが最大のグループの幹/葉の設定が直接使用されます。

6. 落ち葉システム

落ち葉システムは、地面に落ち葉を追加する機能です。葉が描画された後、地面エリアに同じ形状の落ち葉が追加で描画される確率があります。

木の雰囲気を高めます。

6.1 落ち葉密度

落ち葉密度は、葉が地面の落ち葉に変換される確率を決定するパラメータです。値の範囲は 0.0 ~ 1.0 です。値が大きいほど、落ち葉の数が多くなります。

実際の落ち葉の数は、樹冠の密度（木上の葉の数）にも依存します。

6.2 落ち葉の分散度

落ち葉の分散度は、落ち葉の分布範囲を決定するパラメータです。具体的には、元の葉からの水平オフセット範囲（ピクセル）と、地面位置からの垂直オフセット範囲（ピクセル）を指します。値が大きいほど、落ち葉の分布範囲が広くなります。

地面位置からの垂直オフセットには三角分布（2つの均一ランダム値の平均）が使用されます。

元の葉からの水平オフセットには均一分布が使用されます。多数の葉のランダム性が正規分布と同様の効果をすでに達成しているためです。

7. パラメータグループシステム

パラメータグループはフラクタルツリー デスクトップ壁紙の核心的な抽象概念です。単一の木種設定には1つ以上のパラメータグループを含めることができ、各グループは独立したパラメータセットを持ちます。

7.0 パラメータグループが必要な理由

想像してみてください。幹は真っ直ぐで、樹冠は広がった木をデザインしたいとします。

1つのパラメータセットしか使用しない場合、幹と樹冠の形状は同じになり、複雑な効果を実現できません。

実際、これはカエデをデザインした際に私が実際に直面した問題でした。

そこで、私は2つのアイデアを考えました：

最初のアイデアは、枝のy座標を「パラメータのパラメータ」に組み込み、高さに応じてパラメータが変化するように直接実装することです。つまり、パラメータをyの関数として記述できます。

これはプログラマーの思考に合致し、より複雑で正確な形状制御を実現できます。しかし、明らかに直感的ではありません。実際の操作では、かなり複雑な抽象的思考が必要で、計算機やDesmosなどのツールを使って関数グラフを研究し、パラメータを繰り返し調整する必要があります。

そこで、私は2つ目のアイデア、パラメータグループシステムを選びました。

複数のパラメータセットを直接定義し、異なる高さで異なるパラメータグループを適用することで、幹と樹冠の形状を区別します。

実際の開発では、滑らかな移行のために、異なるパラメータグループのパラメータを逆重み付き混合して適用することを選びました。

7.1 基本操作

複数パラメータグループ制御で：

  • ドロップダウン: 編集するパラメータグループを選択します。クリックでパラメータグループ名を変更できます。
  • 「+」: 新しいパラメータグループを追加します（現在選択されているパラメータグループのパラメータをコピー）。
  • 「✕」: 現在選択されているパラメータグループを削除します。

7.2 典型的有効高さ

典型的有効高さは、このパラメータグループのアンカー高さ（ピクセル）で、複数パラメータグループの加重混合計算に使用されます。

パラメータグループが効果を発揮するおおよその位置を制御します。枝が地面からの高さが典型的有効高さに近いほど、パラメータグループの重みが高くなります。

簡単に言うと、パラメータは典型的有効高さ付近で効果を発揮します。

7.3 重み計算

枝が高さ $h$ にある場合、グループ $i$ の元の重みは：

w_i = \min\left(\frac{1}{|h - t_i|},\ 128\right)

ここで $t_i$ はグループ $i$ の典型的な高さです。距離が0の場合、ゼロ除算を避けるため重みは128に固定されます。すべての重みを正規化した後：

  • 通常のスカラーパラメータ（角度範囲、減衰範囲、最小枝、最大深度、葉の閾値、落ち葉パラメータ、幹の長さ、枝の傾斜角）：加重平均。
  • 幹/葉の設定: 最大重みグループの色リストを直接使用。
  • 分布曲線（角度分布、減衰分布）：棒ごとの加重平均。

1つのパラメータグループのみの場合、混合計算をスキップし、そのグループのすべてのパラメータを直接使用します。

7.4 参考例

デフォルトの木種カエデには「幹」と「樹冠」の2つのパラメータグループがあります。両者の唯一の違いは分岐角度範囲、長さ減衰範囲、典型的有効高さです。

「幹」パラメータグループ：

  • 典型的有効高さは 0 で、地面に近いほどパラメータグループの重みが高くなります。
  • 分岐角度範囲は 0 ~ 10 の均一分布で、幹の直立した形状を維持します。
  • 長さ減衰範囲は 0.8 ~ 0.95 の均一分布で、幹が長くなり、下部の枝が減少します。

「樹冠」パラメータグループ：

  • 典型的有効高さは 180 で、基本的に地面から離れるほどパラメータグループの重みが高くなります。
  • 分岐角度範囲は 0 ~ 60 で両端の確率が高く、樹冠のふわふわした形状を維持します。
  • 長さ減衰範囲は 0.3 ~ 0.95 で、大端の確率が高く、小端の確率は低いが可能で、樹冠の層状的な形状を維持し、樹冠の多様性を増加させます。

木種のインポート/エクスポート

プログラムはすべてのカスタム木種を AppData\Local\フラクタルツリー デスクトップ壁紙\trees フォルダの <木の名前>.json ファイルに保存します。パラメータ設定ウィンドウの左下の木のフォルダを開くボタンをクリックして素早く開くことができます。

  • 木種をインポートするには、インポートしたい <木の名前>.json ファイルをそのフォルダに置き、プログラムを再起動してください。
  • 木種をエクスポートするには、エクスポートしたい <木の名前>.json ファイルをコピーしてください。

소개

프랙탈 트리 데스크톱 월페이퍼는 아름다운 나무를 무작위로 생성할 수 있습니다. 강력한 파라미터 시스템 덕분에 자연에 존재하는 나무부터 상상 속의 나무까지 전 세계의 거의 모든 종류의 나무를 시뮬레이션할 수 있습니다.

프랙탈 트리란 무엇인가?

프랙탈 트리는 재귀적으로 생성되는 나무 구조입니다. 각 가지는 여러 하위 가지를 가지고, 각 하위 가지는 또 다시 여러 하위 가지를 포함하며, 이를 반복하여 나무와 같은 구조를 형성합니다.

프랙탈 트리 데스크톱 월페이퍼에서는 렌더링 시 계산량을 줄이고 제어성을 높이기 위해 각 가지가 최대 2개의 하위 가지만 가지는 이진 트리만을 사용합니다.

가능한 한 현실적인 나무 구조를 구현하기 위해 프랙탈 트리 데스크톱 월페이퍼는 노드 간의 거리, 각도, 선 색상 등을 제어하는 다양한 파라미터를 제공합니다.

파라미터 입문

프랙탈 트리 데스크톱 월페이퍼를 처음 사용하시는 분들은 많은 파라미터에 겁먹지 마세요. 몇 가지 핵심 파라미터만 조정하면 독특한 나무를 만들 수 있습니다. 분기 각도 범위와 길이 감쇠 범위를 수정하여 나무의 형태가 어떻게 변하는지 확인해 보세요. 그런 다음 잎 설정의 색상을 조정하여 나무에 색을 더해 보세요. 기본 작업에 익숙해지면 파라미터 그룹과 분포 곡선을 더 탐색하여 각 나무를 진정한 예술 작품으로 만들 수 있습니다.

다음은 기본 파라미터 튜토리얼입니다 ↓

1. 분기 각도 범위

분기 각도 범위는 부모 가지에 대한 자식 가지의 편향 각도 범위(도)를 제어합니다. 분포 곡선을 통해 범위 내에서 각도 값의 분포 확률을 제어합니다(분포 곡선은 기본 튜토리얼 범위를 벗어나므로 기본 균등 분포를 유지하면 됩니다).

  • 분기 각도 범위가 전체적으로 크면 수관은 드문드문하고 볼륨감 있게 보입니다.
  • 분기 각도 범위가 전체적으로 작으면 수관은 빽빽하게 보입니다.
  • 분기 각도 범위의 구간이 넓으면 수관은 무질서하게 보입니다.
  • 분기 각도 범위의 구간이 좁으면 수관은 정돈되게 보입니다.

2. 길이 감쇠 범위

길이 감쇠 범위는 부모 가지 길이에 대한 자식 가지 길이의 비율 범위를 제어합니다. 분포 곡선을 통해 범위 내에서 비율 값의 분포 확률을 제어합니다(이것도 기본 튜토리얼 범위를 벗어나므로 기본 균등 분포를 유지하면 됩니다).

예를 들어, 한 번의 분기에서 길이 감쇠가 사전 설정 범위에서 0.7이 무작위로 선택되면, 자식 가지 길이는 부모 가지 길이의 70%가 됩니다.

  • 길이 감쇠 범위가 전체적으로 크면 수관은 거대하게 보입니다.
  • 길이 감쇠 범위가 전체적으로 작으면 수관은 작고 드문드문하게 보입니다.
  • 길이 감쇠 범위의 구간이 넓으면 수관은 층층이 다양하게 보입니다.
  • 길이 감쇠 범위의 구간이 좁으면 수관은 균일하게 빽빽하게 보입니다.

주의: 특별한 요구 사항이 없다면 길이 감쇠 범위의 최대값은 0.95를 초과하지 않는 것이 좋습니다. 그렇지 않으면 수관이 너무 빽빽해지거나 무한히 성장하여(최대 깊이로만 제한 가능) 렌더링 성능에 영향을 줄 뿐만 아니라 수관이 극히 부자연스럽게 보입니다.

3. 잎 설정

잎 설정은 잎의 형태를 제어합니다. 잎 색상, 가중치, 밀도, 선 너비를 제어하여 다양한 잎 효과를 구현합니다.(기본 튜토리얼에서는 색상만 수정하면 됩니다.)

먼저 약간의 차이가 있는 여러 녹색을 동시에 추가하여 효과를 확인해 보세요.

4. 배경색

이름 그대로 배경색은 캔버스의 배경색을 제어합니다.

연한 잎 색조를 사용하면 더 자연스럽게 보입니다.

더 전문적인 색상 제안을 원하시면 컬러 휠 도구 | Adobe Express를 참조하세요.

파라미터 심화

이 섹션에서는 각 파라미터의 상세한 정의와 사용법을 설명합니다.

픽셀 단위의 모든 파라미터는 창 높이가 표준 1080픽셀일 때의 길이를 나타냅니다. 실제 렌더링 시 창 높이에 따라 비례하여 스케일링됩니다.

1. 글로벌 설정

글로벌 설정은 파라미터 그룹에 포함되지 않고 효과에 직접 영향을 줍니다.

1.1 나무 이름

나무 이름은 나무에 이름을 붙이는 데 사용합니다.

메인 화면 목록에 표시되며, 내보내기 이미지의 기본 파일명 접두사로 사용됩니다. 빈 문자열은 자동으로 "무제"로 수정됩니다.

1.2 배경색

배경색은 캔버스의 배경색을 제어하는 데 사용합니다.

창 배경색과 월페이퍼 이미지 배경색 채우기에 사용됩니다.

2. 기본 형태 파라미터

다음 파라미터는 나무의 기본 형태와 종료 조건을 직접 제어합니다.

2.1 줄기 길이

줄기 길이는 뿌리에서 발하는 첫 번째 레벨 가지의 초기 길이(픽셀)입니다. 뿌리는 캔버스 하단에서 10% 위쪽에 위치합니다.

나무 전체의 기준 스케일을 결정하며, 나무의 최종 크기에 직접 영향을 줍니다.

2.2 최소 가지 길이

최소 가지 길이는 나무의 종료 조건 중 하나입니다. 가지 길이가 이 임계값보다 낮으면 하위 가지가 생성되지 않습니다.

수관 끝의 세밀함과 수관의 크기를 제어합니다.

2.3 최대 깊이

최대 깊이는 나무의 종료 조건 중 하나입니다. 가지의 재귀 레벨(뿌리는 레벨 0, 뿌리에서 분기한 자식 가지는 레벨 1, 이하 동일)이 이 값에 도달하면 분기가 중지됩니다.

수관이 너무 빽빽해지거나 무한히 성장하여 성능에 영향을 주거나 애플리케이션이 멈추는 것을 방지합니다.

주의: 창작 초기에는 최대 깊이를 10~30으로 설정하고, 파라미터가 적절하지 않아 테스트 중 멈추는 것을 방지하기 위해 나중에 필요에 따라 조정하세요.

2.4 잎 임계값

잎 임계값은 현재 가지를 가지로 렌더링할지 잎으로 렌더링할지 결정하는 임계값입니다. 가지 길이에 따라 시각적 역할을 구분합니다. 현재 가지 길이가 이 임계값보다 낮으면 해당 가지는 잎으로 렌더링되고, 임계값보다 높으면 줄기로 렌더링됩니다.

잎의 시각적 효과를 제어하며, 수관의 형태에 영향을 줍니다.

일반적으로 잎 밀도를 직접 조정하는 데 사용하는 것은 권장하지 않습니다. 이는 잎의 형태와 관련이 있으므로 다른 파라미터를 조정하여 구현해야 합니다.

2.5 최대 가지 기울기

최대 가지 기울기는 나무의 종료 조건 중 하나입니다. 자식 가지의 절대 각도(수직 위쪽 방향에 대한 각도)의 상한(도)을 제어합니다. 이 값을 초과하는 자식 가지는 생성되지 않고 직접 종료됩니다.

가지가 아래로 처지는 것을 방지하고 나무의 형태를 유지합니다.

3. 분기 각도 범위

분기 각도 범위는 각 자식 가지가 독립적으로 샘플링하는 편향 각도 구간 범위입니다. 두 자식 가지는 각각 부모 가지의 양쪽으로 편향됩니다. 샘플링 범위는 최소값, 최대값입니다.

가지의 형태를 제어하며, 가지의 정돈됨과 성장 방식에 영향을 줍니다.

3.1 각도 확률 분포 곡선

분포 곡선 버튼으로 막대 그래프 편집기를 엽니다: 막대를 클릭하여 막대 높이를 조정합니다. 높이 범위는 0.02 ~ 1.0입니다.

각 자식 가지의 샘플링 방식: 각도 구간을 8개의 하위 구간으로 균등 분할하고, 먼저 분포 곡선의 막대 그래프 높이를 가중치로 하위 구간을 선택한 후, 해당 하위 구간 내에서 균등하게 무작위 값을 취합니다. 분포 곡선이 지정되지 않으면 전체 구간에서 균등하게 무작위 값을 취합니다.

간단히 말해, 막대가 높을수록 그 부근의 값이 가지 각도로 선택될 확률이 높아집니다.

실제 효과는 소나무가 대부분 곧게 성장하지만 때로는 크게 방향을 틀어, 현실의 소나무의 굽고 힘찬 형태와 유사한 것을 제공합니다.

부모 가지에 평행할 확률이 더 높은 것은 수관의 기본 형태가 너무 혼란스럽지 않도록 유지하기 위함입니다.

4. 길이 감쇠 범위

길이 감쇠 범위는 각 자식 가지가 독립적으로 샘플링하는 길이 감쇠 비율 구간 범위입니다. 자식 가지의 실제 길이 = 부모 가지 길이 × 샘플링된 감쇠 비율입니다. 샘플링 범위는 최소값, 최대값입니다.

4.1 감쇠 확률 분포 곡선

3.1 각도 확률 분포 곡선과 유사합니다.

분포 곡선 버튼으로 막대 그래프 편집기를 엽니다: 막대를 클릭하여 막대 높이를 조정합니다. 높이 범위는 0.02 ~ 1.0입니다.

각 자식 가지의 샘플링 방식: 각도 구간을 8개의 하위 구간으로 균등 분할하고, 먼저 분포 곡선의 막대 그래프 높이를 가중치로 하위 구간을 선택한 후, 해당 하위 구간 내에서 균등하게 무작위 값을 취합니다. 분포 곡선이 지정되지 않으면 전체 구간에서 균등하게 무작위 값을 취합니다.

실제 효과는 녹색 나무가 때로는 가지가 펼쳐지고 때로는 잎을 모아, 층층이 다양한 형태를 제공합니다.

5. 줄기/잎 설정

줄기/잎 설정은 줄기와 잎의 시각적 효과를 결정하는 파라미터 집합으로, 나무의 구조에는 영향을 주지 않습니다.

줄기와 잎은 각각 독립적인 색상 목록을 가집니다. 각 색상 항목은 세 가지 속성을 포함합니다:

5.1 색상

색상은 줄기 또는 잎 렌더링에 사용할 색상을 결정하는 파라미터입니다. 줄기/잎에 여러 색상을 설정할 수 있으며 무작위로 선택됩니다.

5.2 가중치

가중치는 새 줄기 또는 잎을 그릴 때 이 색상이 선택될 확률을 결정하는 파라미터입니다. 값이 클수록 해당 색상의 줄기/잎에서의 비율이 높아집니다.

5.3 선 너비 비율

선 너비 비율은 줄기 또는 잎의 너비 계수를 결정하는 파라미터입니다. 값이 클수록 줄기/잎이 굵어집니다. 실제 굵기는 가지 길이에도 좌우됩니다.

다중 파라미터 그룹 동작: 줄기/잎 설정은 다중 파라미터 그룹의 수치 가중 평균에 참여하지 않습니다. 여러 파라미터 그룹이 존재하면 가중치가 가장 큰 그룹의 줄기/잎 설정을 직접 사용합니다.

6. 낙엽 시스템

낙엽 시스템은 지면에 낙엽을 추가하는 기능입니다. 잎이 그려진 후 지면 영역에 동일한 형태의 낙엽이 추가로 그려질 확률이 있습니다.

나무의 분위기를 높입니다.

6.1 낙엽 밀도

낙엽 밀도는 잎이 지면 낙엽으로 변환될 확률을 결정하는 파라미터입니다. 값 범위는 0.0 ~ 1.0입니다. 값이 클수록 낙엽 수가 많아집니다.

실제 낙엽 수는 수관의 밀도(나무 위의 잎 수)에도 좌우됩니다.

6.2 낙엽 분산도

낙엽 분산도는 낙엽의 분포 범위를 결정하는 파라미터입니다. 구체적으로는 원래 잎에서의 수평 오프셋 범위(픽셀)와 지면 위치에서의 수직 오프셋 범위(픽셀)를 가리킵니다. 값이 클수록 낙엽의 분포 범위가 넓어집니다.

지면 위치에서의 수직 오프셋은 삼각 분포(두 균등 무작위 값의 평균)를 사용합니다.

원래 잎에서의 수평 오프셋은 균등 분포를 사용합니다. 다수의 잎의 무작위성이 이미 정규 분포와 유사한 효과를 달성하기 때문입니다.

7. 파라미터 그룹 시스템

파라미터 그룹은 프랙탈 트리 데스크톱 월페이퍼의 핵심 추상 개념입니다. 단일 나무 종 설정은 하나 이상의 파라미터 그룹을 포함할 수 있으며, 각 그룹은 독립적인 파라미터 세트를 가집니다.

7.0 파라미터 그룹이 필요한 이유

상상해 보세요. 줄기는 곧게 뻗고 수관은 펼쳐진 나무를 디자인하고 싶다고 가정합시다.

하나의 파라미터 세트만 사용하면 줄기와 수관의 형태가 같을 수밖에 없어 복잡한 효과를 구현할 수 없습니다.

사실 이것은 단풍나무를 디자인할 때 제가 실제로 겪었던 문제입니다.

이를 위해 저는 두 가지 아이디어를 생각했습니다:

첫 번째 아이디어는 가지의 y 좌표를 "파라미터의 파라미터"로 통합하여 높이에 따라 파라미터가 변화하도록 직접 구현하는 것입니다. 즉, 파라미터를 y의 함수로 작성할 수 있습니다.

이것은 프로그래머의 사고에 부합하며 더 복잡하고 정확한 형태 제어를 구현할 수 있습니다. 하지만 분명히 직관적이지 않습니다. 실제 작업 시 상당히 복잡한 추상적 사고가 필요하며, 계산기나 Desmos 같은 도구를 사용하여 함수 그래프를 연구하고 파라미터를 반복적으로 조정해야 합니다.

그래서 저는 두 번째 아이디어인 파라미터 그룹 시스템을 선택했습니다.

여러 파라미터 세트를 직접 정의하고 서로 다른 높이에서 서로 다른 파라미터 그룹을 적용하여 줄기와 수관의 형태를 구분합니다.

실제 개발 시 자연스러운 전환을 위해 서로 다른 파라미터 그룹의 파라미터를 역가중 혼합하여 적용하기로 했습니다.

7.1 기본 작업

다중 파라미터 그룹 제어에서:

  • 드롭다운: 편집할 파라미터 그룹을 선택합니다. 클릭하여 파라미터 그룹 이름을 수정할 수 있습니다.
  • 「+」: 새 파라미터 그룹을 추가합니다(현재 선택된 파라미터 그룹의 파라미터 복사).
  • 「✕」: 현재 선택된 파라미터 그룹을 삭제합니다.

7.2 대표적 유효 높이

대표적 유효 높이는 이 파라미터 그룹의 앵커 높이(픽셀)로, 다중 파라미터 그룹의 가중 혼합 계산에 사용됩니다.

파라미터 그룹이 효과를 발휘하는 대략적인 위치를 제어합니다. 가지가 지면에서의 높이가 대표적 유효 높이에 가까울수록 파라미터 그룹의 가중치가 높아집니다.

간단히 말해, 파라미터는 대표적 유효 높이 부근에서 효과를 발휘합니다.

7.3 가중치 계산

가지가 높이 $h$에 있을 때, 그룹 $i$의 원래 가중치는:

w_i = \min\left(\frac{1}{|h - t_i|},\ 128\right)

여기서 $t_i$는 그룹 $i$의 대표적 높이입니다. 거리가 0이면 0으로 나누는 것을 방지하기 위해 가중치는 128로 고정됩니다. 모든 가중치를 정규화한 후:

  • 일반 스칼라 파라미터(각도 범위, 감쇠 범위, 최소 가지, 최대 깊이, 잎 임계값, 낙엽 파라미터, 줄기 길이, 가지 기울기): 가중 평균.
  • 줄기/잎 설정: 최대 가중치 그룹의 색상 목록을 직접 사용.
  • 분포 곡선(각도 분포, 감쇠 분포): 막대별 가중 평균.

하나의 파라미터 그룹만 있으면 혼합 계산을 건너뛰고 해당 그룹의 모든 파라미터를 직접 사용합니다.

7.4 참고 예시

기본 나무 종 단풍나무에는 "줄기"와 "수관" 두 개의 파라미터 그룹이 있습니다. 양자의 유일한 차이는 분기 각도 범위, 길이 감쇠 범위, 대표적 유효 높이입니다.

"줄기" 파라미터 그룹:

  • 대표적 유효 높이는 0이며, 지면에 가까울수록 파라미터 그룹의 가중치가 높아집니다.
  • 분기 각도 범위는 0 ~ 10 균등 분포로 줄기의 직립 형태를 유지합니다.
  • 길이 감쇠 범위는 0.8 ~ 0.95 균등 분포로 줄기가 길게 유지되고 하부 가지가 감소합니다.

"수관" 파라미터 그룹:

  • 대표적 유효 높이는 180이며, 기본적으로 지면에서 멀어질수록 파라미터 그룹의 가중치가 높아집니다.
  • 분기 각도 범위는 0 ~ 60으로 양단의 확률이 높아 수관의 볼륨감 있는 형태를 유지합니다.
  • 길이 감쇠 범위는 0.3 ~ 0.95로 대단의 확률이 높고 소단의 확률은 낮지만 가능하여, 수관의 층층이 다양한 형태를 유지하고 수관의 다양성을 증가시킵니다.

Fractal Tree Fondos de Escritorio - Guía de Usuario

Nota: Este documento ha sido traducido por IA desde la versión original en chino. Si encuentra errores o expresiones poco claras, consulte el archivo original help.md.

Introducción

Fractal Tree Fondos de Escritorio puede generar aleatoriamente un árbol hermoso. Gracias a su potente sistema de parámetros, puede simular casi cualquier tipo de árbol en el mundo, incluyendo aquellos que existen en la naturaleza y aquellos que solo existen en tu imaginación.

¿Qué es un Árbol Fractal?

Un Árbol Fractal es una estructura de árbol generada recursivamente — cada rama tiene múltiples sub-ramas, cada sub-rama contiene múltiples sub-ramas, y así sucesivamente, resultando en una estructura tipo árbol.

En Fractal Tree Fondos de Escritorio, para reducir la computación de renderizado y mejorar la controlabilidad, solo usamos árboles binarios, lo que significa que cada rama tiene máximo dos sub-ramas.

Para lograr una estructura de árbol lo más realista posible, Fractal Tree Fondos de Escritorio proporciona numerosos parámetros para controlar la distancia y el ángulo entre nodos, el color de las líneas, etc.

Introducción a los Parámetros

Si eres nuevo en Fractal Tree Fondos de Escritorio, no te intimides por la gran cantidad de parámetros — solo necesitas ajustar unos pocos parámetros clave para crear un árbol único. Intenta modificar el Rango de Ángulo de Bifurcación y el Rango de Decaimiento de Longitud para ver cómo cambia la forma del árbol; luego ajusta el color en Configuración de Hojas para agregar color a tu árbol. Una vez familiarizado con las operaciones básicas, puedes explorar más a fondo los Grupos de Parámetros y las Curvas de Distribución para hacer cada árbol una verdadera obra de arte.

A continuación se presenta un tutorial básico de parámetros ↓

1. Rango de Ángulo de Bifurcación

Rango de Ángulo de Bifurcación controla el rango del ángulo de desviación (en grados) de las sub-ramas respecto a su rama padre. La curva de distribución controla la probabilidad de los valores de ángulo dentro del rango (por supuesto, las curvas de distribución están fuera del alcance de este tutorial básico, así que mantén la distribución uniforme por defecto).

  • Cuando el Rango de Ángulo de Bifurcación es generalmente grande, la copa del árbol aparece dispersa y esponjosa.
  • Cuando el Rango de Ángulo de Bifurcación es generalmente pequeño, la copa del árbol aparece densa y compacta.
  • Cuando el Rango de Ángulo de Bifurcación tiene un intervalo amplio, la copa del árbol aparece caótica y desordenada.
  • Cuando el Rango de Ángulo de Bifurcación tiene un intervalo estrecho, la copa del árbol aparece ordenada y uniforme.

2. Rango de Decaimiento de Longitud

Rango de Decaimiento de Longitud controla el rango de la proporción de la longitud de la sub-rama respecto a la rama padre. La curva de distribución controla la probabilidad de los valores de proporción dentro del rango (de nuevo, las curvas de distribución están fuera del alcance de este tutorial básico, así que mantén la distribución uniforme por defecto).

Por ejemplo, si una bifurcación obtiene un valor aleatorio de 0.7 del rango preestablecido, la longitud de la sub-rama será el 70% de la longitud de la rama padre.

  • Cuando el Rango de Decaimiento de Longitud es generalmente grande, la copa del árbol aparece enorme.
  • Cuando el Rango de Decaimiento de Longitud es generalmente pequeño, la copa del árbol aparece pequeña y dispersa.
  • Cuando el Rango de Decaimiento de Longitud tiene un intervalo amplio, la copa del árbol aparece variada y en capas.
  • Cuando el Rango de Decaimiento de Longitud tiene un intervalo estrecho, la copa del árbol aparece uniforme y densa.

Ten en cuenta que a menos que tengas requisitos especiales, el valor máximo del Rango de Decaimiento de Longitud no debe exceder 0.95, de lo contrario la copa del árbol será demasiado densa, o incluso crecerá infinitamente (limitado solo por la Profundidad Máxima), lo que no solo afecta el rendimiento de renderizado sino que también hace que la copa del árbol se vea extremadamente antinatural.

3. Configuración de Hojas

Configuración de Hojas controla la apariencia de las hojas. Al ajustar el color, peso, densidad y ancho de línea de las hojas, puedes lograr diferentes efectos de hojas. (Para el tutorial básico, solo modifica los colores.)

Puedes comenzar agregando varios tonos de verde con alguna variación para ver el efecto.

4. Color de Fondo

Como su nombre indica, Color de Fondo controla el color de fondo del lienzo.

Puedes usar un tono claro similar al de las hojas para un aspecto más natural.

Para sugerencias de color más profesionales, puedes consultar Herramienta de Círculo de Color – Adobe Express.

Parámetros Avanzados

Esta sección proporciona definiciones detalladas y el uso de cada parámetro.

Todos los parámetros en píxeles representan longitudes cuando la altura de la ventana es de 1080 píxeles estándar. El renderizado real escala proporcionalmente según la altura de la ventana.

1. Configuración Global

La configuración global no está incluida en ningún grupo de parámetros y afecta directamente el resultado.

1.1 Nombre del Árbol

Nombre del Árbol se usa para nombrar tu árbol.

Se usa para mostrar en la lista de la interfaz principal y como prefijo del nombre de archivo por defecto para las imágenes exportadas. Las cadenas vacías se corrigen automáticamente a "Sin título".

1.2 Color de Fondo

Color de Fondo controla el color de fondo del lienzo.

Se usa para rellenar el color de fondo de la ventana y el color de fondo de la imagen de fondo de pantalla.

2. Parámetros de Forma Básica

Los siguientes parámetros controlan directamente la forma básica y las condiciones de terminación del árbol.

2.1 Longitud del Tronco

Longitud del Tronco es la longitud inicial (en píxeles) de la primera rama que emana de la raíz del árbol. La raíz del árbol se posiciona al 10% arriba del fondo del lienzo.

Se usa para determinar la escala base de todo el árbol, afectando directamente el tamaño final del árbol.

2.2 Longitud Mínima de Rama

Longitud Mínima de Rama es una de las condiciones de terminación del árbol. Cuando la longitud de una rama cae por debajo de este umbral, no se generan más sub-ramas.

Se usa para controlar la densidad de los extremos de la copa del árbol y el tamaño de la copa.

2.3 Profundidad Máxima

Profundidad Máxima es una de las condiciones de terminación del árbol. Cuando el nivel de recursión de una rama (la raíz es nivel 0, las sub-ramas de la raíz son nivel 1, y así sucesivamente) alcanza este valor, la bifurcación se detiene.

Se usa para evitar que la copa del árbol sea demasiado densa o crezca infinitamente, lo que afectaría el rendimiento o incluso congelaría la aplicación.

Ten en cuenta que en las etapas iniciales de creación, se recomienda establecer la Profundidad Máxima en 10 ~ 30, y ajustar más tarde según sea necesario para evitar congelaciones durante las pruebas debido a parámetros poco razonables.

2.4 Umbral de Hoja

Umbral de Hoja determina si la rama actual debe renderizarse como una ramita o como una hoja. Basado en la longitud de la rama para determinar el rol visual. Cuando la longitud de la rama actual está por debajo de este umbral, la rama se renderiza como una hoja; por encima del umbral, se renderiza como un tronco.

Se usa para controlar el efecto visual de las hojas, afectando la forma de la copa del árbol.

Generalmente no se recomienda usar esto para ajustar directamente la densidad de las hojas, ya que esto está relacionado con la forma de las hojas. Se deben ajustar otros parámetros en su lugar.

2.5 Inclinación Máxima de Rama

Inclinación Máxima de Rama es una de las condiciones de terminación del árbol. Controla el límite superior (en grados) del ángulo absoluto de las sub-ramas (el ángulo relativo a la dirección vertical hacia arriba). Cualquier sub-rama cuyo ángulo exceda este valor no se creará y terminará directamente.

Se usa para evitar que las ramas caigan y mantener la forma del árbol.

3. Rango de Ángulo de Bifurcación

Rango de Ángulo de Bifurcación es el rango del intervalo del ángulo de desviación muestreado independientemente para cada sub-rama. Dos sub-ramas se desvían a cada lado de la rama padre respectivamente. El rango de muestreo es [Mínimo, Máximo] (grados).

Se usa para controlar la forma de las ramas, afectando la uniformidad y el patrón de crecimiento de las ramas.

3.1 Curva de Distribución de Probabilidad de Ángulo

Abre el editor de gráficos de barras a través del botón Curva de Distribución: haz clic en las barras para ajustar las alturas de las barras, con un rango de altura de 0.02 ~ 1.0.

Método de muestreo para cada sub-rama: Divide el intervalo de ángulo en 8 sub-intervalos iguales, primero selecciona un sub-intervalo usando las alturas de las barras de la curva de distribución como pesos, luego toma un valor aleatorio uniforme dentro de ese sub-intervalo. Si no se especifica una curva de distribución, se toma un valor aleatorio uniforme del intervalo global.

En pocas palabras, cuanto más alta sea la barra, mayor será la probabilidad de que los valores cercanos sean seleccionados como ángulos de ramas.

3.2 Ejemplo de Referencia

El árbol por defecto Pino tiene un rango de ángulo de bifurcación de [0, 90], con una curva de distribución de probabilidad de ángulo aproximadamente como sigue:

Se puede ver que los ángulos de bifurcación del pino tienen más probabilidad de estar alrededor de 0 y 90 grados. Es decir, las ramas del pino tienen más probabilidad de ser casi paralelas o perpendiculares a la rama padre, en lugar de crecer diagonalmente.

El efecto real es que los pinos mayormente crecen rectos, pero a veces hacen grandes giros, proporcionando una forma similar a la retorcida y vigorosa de los pinos reales.

Entre ellos, es más probable que sean paralelos a la rama padre para mantener la forma básica de la copa del árbol sin volverse demasiado caótica.

4. Rango de Decaimiento de Longitud

Rango de Decaimiento de Longitud es el rango del intervalo de proporción de decaimiento de longitud muestreado independientemente para cada sub-rama. La longitud real de la sub-rama = longitud de la rama padre × proporción de decaimiento muestreada. El rango de muestreo es [Mínimo, Máximo] (proporción de decaimiento).

4.1 Curva de Distribución de Probabilidad de Decaimiento

Similar a 3.1 Curva de Distribución de Probabilidad de Ángulo.

Abre el editor de gráficos de barras a través del botón Curva de Distribución: haz clic en las barras para ajustar las alturas de las barras, con un rango de altura de 0.02 ~ 1.0.

Método de muestreo para cada sub-rama: Divide el intervalo de ángulo en 8 sub-intervalos iguales, primero selecciona un sub-intervalo usando las alturas de las barras de la curva de distribución como pesos, luego toma un valor aleatorio uniforme dentro de ese sub-intervalo. Si no se especifica una curva de distribución, se toma un valor aleatorio uniforme del intervalo global.

En pocas palabras, cuanto más alta sea la barra, mayor será la probabilidad de que los valores cercanos sean seleccionados como proporciones de decaimiento.

4.2 Ejemplo de Referencia

El árbol por defecto Árbol Verde tiene un rango de decaimiento de longitud de [0.6, 0.94], con una curva de distribución de probabilidad de decaimiento aproximadamente como sigue:

0.60
0.65
0.70
0.75
0.79
0.84
0.89
0.94

Se puede ver que las proporciones de decaimiento del árbol verde tienen más probabilidad de estar alrededor de 0.60 y 0.94. Es decir, las ramas del árbol verde tienen casi la misma longitud que la rama padre o notablemente más cortas.

El efecto real es que el árbol verde a veces tiene ramas extendidas y a veces hojas recogidas, proporcionando una forma en capas y variada.

Vale la pena señalar que si el árbol verde elige una rama corta demasiado pronto, reducirá directamente la altura de todo el árbol, resultando en una mayor variación en la altura del árbol.

5. Configuración de Tronco/Hojas

Configuración de Tronco/Hojas es una colección de parámetros que determinan el efecto visual de troncos y hojas, sin afectar la estructura del árbol.

Los troncos y las hojas tienen listas de colores independientes. Cada entrada de color contiene tres atributos:

5.1 Color

Color determina el color usado para renderizar troncos u hojas. Soporta establecer múltiples colores para troncos/hojas, seleccionados aleatoriamente.

5.2 Peso

Peso determina la probabilidad de que este color sea seleccionado al dibujar un nuevo tronco u hoja. Cuanto mayor sea el valor, mayor será la proporción de este color en troncos/hojas.

5.3 Proporción de Ancho de Línea

Proporción de Ancho de Línea determina el coeficiente de ancho de troncos u hojas. Cuanto mayor sea el valor, más grueso será el tronco/hoja. El grosor real también depende de la longitud de la rama.

Comportamiento de Múltiples Grupos de Parámetros: La configuración de tronco/hojas no participa en el promedio ponderado numérico de múltiples grupos de parámetros. Cuando existen múltiples grupos de parámetros, se usa directamente la lista de colores del grupo con el mayor peso.

6. Sistema de Hojas Caídas

Sistema de Hojas Caídas es una función para agregar hojas caídas en el suelo. Después de que se dibuja una hoja, hay una probabilidad de dibujar una hoja caída adicional con la misma forma en el área del suelo.

Se usa para mejorar la atmósfera del árbol.

6.1 Densidad de Hojas Caídas

Densidad de Hojas Caídas determina la probabilidad de que las hojas se conviertan en hojas caídas en el suelo. El rango de valores es 0.0 ~ 1.0. Cuanto mayor sea el valor, más hojas caídas habrá.

El número real de hojas caídas también depende de la densidad de la copa del árbol (el número de hojas en el árbol).

6.2 Dispersión de Hojas Caídas

Dispersión de Hojas Caídas determina el rango de distribución de las hojas caídas, específicamente el rango de desplazamiento horizontal (en píxeles) relativo a la hoja original y el rango de desplazamiento vertical (en píxeles) relativo a la posición del suelo. Cuanto mayor sea el valor, mayor será el rango de distribución de las hojas caídas.

El desplazamiento vertical relativo a la posición del suelo usa una distribución triangular (promedio de dos valores aleatorios uniformes).

El desplazamiento horizontal relativo a la hoja original usa una distribución uniforme, porque la aleatoriedad de una gran cantidad de hojas ya logra un efecto similar a una distribución normal.

7. Sistema de Grupos de Parámetros

Grupo de Parámetros es el concepto de abstracción central de Fractal Tree Fondos de Escritorio. Una configuración de árbol único puede contener uno o más grupos de parámetros, cada grupo con un conjunto independiente de parámetros.

7.0 Por qué se Necesitan Grupos de Parámetros

Imagina que quieres diseñar un árbol con un tronco recto pero una copa extendida.

Si solo usas un conjunto de parámetros, el tronco y la copa solo pueden tener la misma forma, lo que hace imposible lograr efectos complejos.

De hecho, este fue el problema real que encontré al diseñar Arce.

Para esto, tuve dos ideas:

La primera idea era incorporar la coordenada y de la rama en "parámetros de parámetros", lo que podía implementar directamente parámetros que cambian con la altura. En otras palabras, los parámetros se podían escribir como funciones de y.

Esto se alinea bien con el pensamiento del programador y puede lograr un control de forma más complejo y preciso. Sin embargo, es claramente poco intuitivo. En la práctica, requiere un pensamiento abstracto bastante complejo, y necesita calculadoras o incluso herramientas como Desmos para estudiar gráficos de funciones y ajustar parámetros repetidamente.

Por lo tanto, elegí la segunda idea: el sistema de grupos de parámetros.

Al definir directamente múltiples conjuntos de parámetros y aplicar diferentes grupos de parámetros a diferentes alturas, se pueden distinguir las formas del tronco y la copa.

Durante el desarrollo real, para transiciones suaves, elegí aplicar una mezcla ponderada inversa de parámetros de diferentes grupos de parámetros.

7.1 Operaciones Básicas

En Control de Múltiples Grupos de Parámetros:

  • Desplegable: Selecciona un grupo de parámetros para editar. Haz clic para modificar el nombre del grupo de parámetros.
  • 「+」: Agrega un nuevo grupo de parámetros (copia los parámetros del grupo de parámetros actualmente seleccionado).
  • 「✕」: Elimina el grupo de parámetros actualmente seleccionado.

7.2 Altura Efectiva Típica

Altura Efectiva Típica es la altura de anclaje (en píxeles) de este grupo de parámetros, usada para los cálculos de mezcla ponderada de múltiples grupos de parámetros.

Se usa para controlar la posición aproximada donde el grupo de parámetros tiene efecto. Cuanto más cerca esté la rama de la altura efectiva típica desde el suelo, mayor será el peso del grupo de parámetros.

En pocas palabras, los parámetros tendrán efecto cerca de la Altura Efectiva Típica.

7.3 Cálculo de Peso

Cuando una rama está a la altura $h$, el peso original del grupo $i$ es:

w_i = \min\left(\frac{1}{|h - t_i|},\ 128\right)

Donde $t_i$ es la altura típica del grupo $i$. El peso se fija en 128 cuando la distancia es 0 para evitar la división por cero. Después de normalizar todos los pesos:

  • Parámetros escalares ordinarios (rango de ángulo, rango de decaimiento, rama mínima, profundidad máxima, umbral de hoja, parámetros de hojas caídas, longitud del tronco, inclinación de rama): Promedio ponderado.
  • Configuración de Tronco/Hojas: Usar directamente la lista de colores del grupo de mayor peso.
  • Curvas de Distribución (distribución de ángulo, distribución de decaimiento): Promedio ponderado por barra.

Cuando solo hay un grupo de parámetros, se omite el cálculo de mezcla y se usan directamente todos los parámetros de ese grupo.

7.4 Ejemplo de Referencia

El árbol por defecto Arce tiene dos grupos de parámetros: "Tronco" y "Copa". La única diferencia entre ellos es el Rango de Ángulo de Bifurcación, el Rango de Decaimiento de Longitud y la Altura Efectiva Típica.

Grupo de parámetros "Tronco":

  • La altura efectiva típica es 0, lo que significa que cuanto más cerca del suelo, mayor es el peso del grupo de parámetros.
  • El rango de ángulo de bifurcación es 0 ~ 10 distribución uniforme, manteniendo la forma erguida del tronco.
  • El rango de decaimiento de longitud es 0.8 ~ 0.95 distribución uniforme, asegurando que el tronco sea largo y reduciendo las ramas inferiores.

Grupo de parámetros "Copa":

  • La altura efectiva típica es 180, básicamente cuanto más lejos del suelo, mayor es el peso del grupo de parámetros.
  • El rango de ángulo de bifurcación es 0 ~ 60 con alta probabilidad en ambos extremos, manteniendo la forma esponjosa de la copa.
  • El rango de decaimiento de longitud es 0.3 ~ 0.95 con alta probabilidad en el extremo grande y baja pero posible probabilidad en el extremo pequeño, manteniendo la forma en capas de la copa y aumentando la diversidad de la copa.

Importar/Exportar Especies de Árboles

El programa guarda todas las especies de árboles personalizadas en archivos .json en la carpeta AppData\Local\Fractal Tree Fondos de Escritorio\trees. Puedes hacer clic en el botón Abrir Carpeta de Árboles en la parte inferior izquierda de la ventana de configuración de parámetros para abrirlo rápidamente.

  • Para importar una especie de árbol, simplemente coloca el archivo .json que deseas importar en esa carpeta y reinicia el programa.
  • Para exportar una especie de árbol, simplemente copia el archivo .json que deseas exportar.

Ventana de Renderizado

Haz clic en cualquier botón de especie de árbol en la interfaz principal o en Vista Previa en la interfaz de edición para abrir la ventana de renderizado del árbol fractal. Soporta arrastrar y hacer zoom.

  • Actualizar: Reinicia la semilla aleatoria y vuelve a dibujar. Cada clic produce una forma diferente.
  • Guardar como Imagen: Renderiza a PNG al tamaño de la ventana, guarda en el escritorio como _.png. La ventana permanece abierta después de completar.
  • Establecer como Fondo de Escritorio: Renderiza a BMP a 2x la resolución de pantalla, llama a la API del sistema para establecer como fondo de escritorio de Windows. La ventana se cierra automáticamente después de completar.

Einführung

Fraktalbaum Desktop-Hintergrundbild kann zufällig einen schönen Baum generieren. Dank seines leistungsstarken Parametersystems kann es fast jede Art von Baum in der Welt simulieren, einschließlich derer, die in der Natur existieren, und derer, die nur in Ihrer Vorstellung existieren.

Was ist ein Fraktalbaum?

Ein Fraktalbaum ist eine rekursiv generierte Baumstruktur — jeder Ast hat mehrere Unteräste, jeder Unterast enthält mehrere Unteräste, und so weiter, was zu einer baumartigen Struktur führt.

In Fraktalbaum Desktop-Hintergrundbild verwenden wir nur Binärbäume, um die Rendering-Berechnung zu reduzieren und die Kontrollierbarkeit zu verbessern, was bedeutet, dass jeder Ast maximal zwei Unteräste hat.

Um eine möglichst realistische Baumstruktur zu erreichen, bietet Fraktalbaum Desktop-Hintergrundbild zahlreiche Parameter zur Kontrolle des Abstands und Winkels zwischen Knoten, der Linienfarben usw.

Parameter-Einführung

Wenn Sie neu bei Fraktalbaum Desktop-Hintergrundbild sind, lassen Sie sich nicht von der großen Anzahl an Parametern einschüchtern — Sie müssen nur einige Kernparameter anpassen, um einen einzigartigen Baum zu erstellen. Versuchen Sie, den Verzweigungswinkelbereich und den Längenabnahmebereich zu ändern, um zu sehen, wie sich die Baumform verändert; passen Sie dann die Farbe in den Blatteinstellungen an, um Ihrem Baum Farbe zu verleihen. Wenn Sie mit den grundlegenden Operationen vertraut sind, können Sie weiter die Parametergruppen und Verteilungskurven erkunden, um jeden Baum zu einem wahren Kunstwerk zu machen.

Im Folgenden finden Sie ein grundlegendes Parameter-Tutorial ↓

1. Verzweigungswinkelbereich

Verzweigungswinkelbereich steuert den Bereich des Ablenkungswinkels (in Grad) der Unteräste relativ zum Elternast. Die Verteilungskurve steuert die Wahrscheinlichkeit der Winkelwerte im Bereich (natürlich sind Verteilungskurven nicht Teil dieses grundlegenden Tutorials, behalten Sie also die Standard-Gleichverteilung bei).

  • Wenn der Verzweigungswinkelbereich generell groß ist, erscheint die Baumkrone dünn und flauschig.
  • Wenn der Verzweigungswinkelbereich generell klein ist, erscheint die Baumkrone dicht und kompakt.
  • Wenn der Verzweigungswinkelbereich einen breiten Intervall hat, erscheint die Baumkrone chaotisch und unordentlich.
  • Wenn der Verzweigungswinkelbereich einen engen Intervall hat, erscheint die Baumkrone ordentlich und gleichmäßig.

2. Längenabnahmebereich

Längenabnahmebereich steuert den Bereich des Verhältnisses der Unterastlänge zur Elternastlänge. Die Verteilungskurve steuert die Wahrscheinlichkeit der Verhältniswerte im Bereich (auch dies ist nicht Teil dieses grundlegenden Tutorials, behalten Sie also die Standard-Gleichverteilung bei).

Wenn zum Beispiel eine Verzweigung einen zufälligen Wert von 0,7 aus dem voreingestellten Bereich erhält, wird die Unterastlänge 70% der Elternastlänge betragen.

  • Wenn der Längenabnahmebereich generell groß ist, erscheint die Baumkrone riesig.
  • Wenn der Längenabnahmebereich generell klein ist, erscheint die Baumkrone klein und dünn.
  • Wenn der Längenabnahmebereich einen breiten Intervall hat, erscheint die Baumkrone abgestuft und vielfältig.
  • Wenn der Längenabnahmebereich einen engen Intervall hat, erscheint die Baumkrone gleichmäßig und dicht.

Beachten Sie, dass der Maximalwert des Längenabnahmebereichs ohne besondere Anforderungen 0,95 nicht überschreiten sollte, sonst wird die Baumkrone zu dicht oder wächst sogar unendlich (nur durch die Maximale Tiefe begrenzt), was nicht nur die Rendering-Leistung beeinträchtigt, sondern auch die Baumkrone extrem unnatürlich aussehen lässt.

3. Blatteinstellungen

Blatteinstellungen steuert das Erscheinungsbild der Blätter. Durch Anpassung von Blattfarbe, Gewicht, Dichte und Linienbreite können Sie verschiedene Blatteffekte erzielen. (Für das grundlegende Tutorial genügt es, die Farben zu ändern.)

Sie können damit beginnen, mehrere Grüntöne mit gewisser Variation hinzuzufügen, um den Effekt zu sehen.

4. Hintergrundfarbe

Wie der Name schon sagt, steuert Hintergrundfarbe die Hintergrundfarbe der Leinwand.

Sie können einen hellen, blattähnlichen Farbton für ein natürlicheres Aussehen verwenden.

Für professionellere Farbvorschläge können Sie das Farbrad-Werkzeug | Adobe Express konsultieren.

Fortgeschrittene Parameter

Dieser Abschnitt bietet detaillierte Definitionen und Verwendung jedes Parameters.

Alle Parameter in Pixeln repräsentieren Längen bei einer Standardfensterhöhe von 1080 Pixeln. Das tatsächliche Rendering skaliert proportional basierend auf der Fensterhöhe.

1. Globale Einstellungen

Die globalen Einstellungen sind in keiner Parametergruppe enthalten und beeinflussen das Ergebnis direkt.

1.1 Baumname

Baumname wird verwendet, um Ihrem Baum einen Namen zu geben.

Er wird in der Liste der Hauptschnittstelle angezeigt und als Standard-Dateinamenpräfix für exportierte Bilder verwendet. Leere Zeichenfolgen werden automatisch zu "Unbenannt" korrigiert.

1.2 Hintergrundfarbe

Hintergrundfarbe wird verwendet, um die Hintergrundfarbe der Leinwand zu steuern.

Sie wird zum Füllen der Fensterhintergrundfarbe und der Hintergrundfarbe des Hintergrundbildes verwendet.

2. Grundlegende Formparameter

Die folgenden Parameter steuern direkt die Grundform und die Abbruchbedingungen des Baumes.

2.1 Stammlänge

Stammlänge ist die Anfangslänge (in Pixeln) des ersten Astes, der von der Wurzel ausgeht. Die Wurzel befindet sich 10% über dem unteren Rand der Leinwand.

Sie bestimmt die Basisskala des gesamten Baumes und beeinflusst direkt die endgültige Größe des Baumes.

2.2 Minimale Astlänge

Minimale Astlänge ist eine der Abbruchbedingungen des Baumes. Wenn die Astlänge unter diesen Schwellenwert fällt, werden keine weiteren Unteräste generiert.

Sie steuert die Feinheit der Enden der Baumkrone und die Größe der Krone.

2.3 Maximale Tiefe

Maximale Tiefe ist eine der Abbruchbedingungen des Baumes. Wenn die Rekursionsebene eines Astes (die Wurzel ist Ebene 0, Unteräste von der Wurzel sind Ebene 1 usw.) diesen Wert erreicht, wird die Verzweigung gestoppt.

Sie verhindert, dass die Baumkrone zu dicht wird oder unendlich wächst, was die Leistung beeinträchtigt oder die Anwendung sogar einfrieren lässt.

Beachten Sie, dass in den frühen Phasen der Erstellung die Maximale Tiefe auf 10 ~ 30 eingestellt werden sollte, und später je nach Bedarf angepasst werden sollte, um Einfrierungen während des Testens aufgrund unangemessener Parameter zu vermeiden.

2.4 Blattschwellenwert

Blattschwellenwert bestimmt, ob der aktuelle Ast als Zweig oder als Blatt gerendert werden soll. Basierend auf der Astlänge wird die visuelle Rolle bestimmt. Wenn die aktuelle Astlänge unter diesem Schwellenwert liegt, wird der Ast als Blatt gerendert; über dem Schwellenwert wird er als Stamm gerendert.

Er steuert den visuellen Effekt der Blätter und beeinflusst die Form der Baumkrone.

Generell wird nicht empfohlen, dies direkt zur Anpassung der Blattdichte zu verwenden, da dies mit der Blattform zusammenhängt. Andere Parameter sollten stattdessen angepasst werden.

2.5 Maximale Astneigung

Maximale Astneigung ist eine der Abbruchbedingungen des Baumes. Sie steuert die Obergrenze (in Grad) des absoluten Winkels der Unteräste (der Winkel relativ zur vertikal aufwärts gerichteten Richtung). Jeder Unterast, dessen Winkel diesen Wert überschreitet, wird nicht erstellt und endet direkt.

Sie verhindert, dass Äste herabhängen, und erhält die Form des Baumes.

3. Verzweigungswinkelbereich

Verzweigungswinkelbereich ist der Intervallbereich des Ablenkungswinkels, der für jeden Unterast unabhängig abgetastet wird. Zwei Unteräste weichen jeweils zu beiden Seiten des Elternastes ab. Der Abtastbereich ist [Minimum, Maximum] (Grad).

Er steuert die Form der Äste und beeinflusst die Gleichmäßigkeit und das Wachstumsmuster der Äste.

3.1 Winkelwahrscheinlichkeits-Verteilungskurve

Öffnen Sie den Balkendiagramm-Editor über die Schaltfläche Verteilungskurve: Klicken Sie auf die Balken, um die Balkenhöhen anzupassen. Der Höhenbereich ist 0,02 ~ 1,0.

Abtastmethode für jeden Unterast: Teilen Sie den Winkelintervall in 8 gleiche Unterintervalle, wählen Sie zuerst ein Unterintervall mit den Balkenhöhen der Verteilungskurve als Gewichte, und nehmen Sie dann einen gleichmäßig zufälligen Wert innerhalb dieses Unterintervalls. Wenn keine Verteilungskurve angegeben ist, wird ein gleichmäßig zufälliger Wert aus dem globalen Intervall genommen.

Einfach ausgedrückt: Je höher der Balken, desto höher ist die Wahrscheinlichkeit, dass nahegelegene Werte als Astwinkel ausgewählt werden.

3.2 Referenzbeispiel

Der Standardbaum Kiefer hat einen Verzweigungswinkelbereich von [0, 90], mit einer Winkelwahrscheinlichkeits-Verteilungskurve ungefähr wie folgt:

Man sieht, dass die Verzweigungswinkel der Kiefer mit höherer Wahrscheinlichkeit um 0 und 90 Grad liegen. Das heißt, die Äste der Kiefer sind eher fast parallel oder senkrecht zum Elternast, statt diagonal zu wachsen.

Der tatsächliche Effekt ist, dass Kiefern meist gerade wachsen, aber manchmal große Richtungsänderungen machen, was eine Form ähnlich der verdrehten und kräftigen Form echter Kiefern bietet.

Dabei ist eine größere Wahrscheinlichkeit, parallel zum Elternast zu sein, um die Grundform der Baumkrone nicht zu chaotisch zu halten.

4. Längenabnahmebereich

Längenabnahmebereich ist der Intervallbereich des Längenabnahmeverhältnisses, der für jeden Unterast unabhängig abgetastet wird. Die tatsächliche Länge des Unterastes = Länge des Elternastes × abgetastetes Abnahmeverhältnis. Der Abtastbereich ist [Minimum, Maximum] (Abnahmeverhältnis).

4.1 Abnahmewahrscheinlichkeits-Verteilungskurve

Ähnlich wie 3.1 Winkelwahrscheinlichkeits-Verteilungskurve.

Öffnen Sie den Balkendiagramm-Editor über die Schaltfläche Verteilungskurve: Klicken Sie auf die Balken, um die Balkenhöhen anzupassen. Der Höhenbereich ist 0,02 ~ 1,0.

Abtastmethode für jeden Unterast: Teilen Sie den Winkelintervall in 8 gleiche Unterintervalle, wählen Sie zuerst ein Unterintervall mit den Balkenhöhen der Verteilungskurve als Gewichte, und nehmen Sie dann einen gleichmäßig zufälligen Wert innerhalb dieses Unterintervalls. Wenn keine Verteilungskurve angegeben ist, wird ein gleichmäßig zufälliger Wert aus dem globalen Intervall genommen.

Einfach ausgedrückt: Je höher der Balken, desto höher ist die Wahrscheinlichkeit, dass nahegelegene Werte als Abnahmeverhältnis ausgewählt werden.

4.2 Referenzbeispiel

Der Standardbaum Grüner Baum hat einen Längenabnahmebereich von [0,6, 0,94], mit einer Abnahmewahrscheinlichkeits-Verteilungskurve ungefähr wie folgt:

Der tatsächliche Effekt ist, dass der grüne Baum manchmal ausgebreitete Äste hat und manchmal eingesammelte Blätter, was eine geschichtete und vielfältige Form bietet.

Es ist erwähnenswert, dass wenn der grüne Baum zu früh einen kurzen Ast wählt, dies direkt die Höhe des gesamten Baumes reduziert, was zu einer größeren Variation der Baumhöhe führt.

5. Stamm/Blatt-Einstellungen

Stamm/Blatt-Einstellungen ist eine Sammlung von Parametern, die den visuellen Effekt von Stämmen und Blättern bestimmen, ohne die Baumstruktur zu beeinflussen.

Stämme und Blätter haben jeweils unabhängige Farblisten. Jeder Farbeingang enthält drei Attribute:

5.1 Farbe

Farbe ist ein Parameter, der die zum Rendern von Stämmen oder Blättern verwendete Farbe bestimmt. Es werden mehrere Farben für Stämme/Blätter unterstützt, die zufällig ausgewählt werden.

5.2 Gewicht

Gewicht ist ein Parameter, der die Wahrscheinlichkeit bestimmt, dass diese Farbe beim Zeichnen eines neuen Stammes oder Blattes ausgewählt wird. Je größer der Wert, desto höher der Anteil dieser Farbe in Stämmen/Blättern.

5.3 Linienbreitenverhältnis

Linienbreitenverhältnis ist ein Parameter, der den Breitenkoeffizienten von Stämmen oder Blättern bestimmt. Je größer der Wert, desto dicker wird der Stamm/das Blatt. Die tatsächliche Dicke hängt auch von der Astlänge ab.

Verhalten bei mehreren Parametergruppen: Die Stamm/Blatt-Einstellungen nehmen nicht am numerischen gewichteten Durchschnitt mehrerer Parametergruppen teil. Wenn mehrere Parametergruppen existieren, wird die Farbliste der Parametergruppe mit dem größten Gewicht direkt verwendet.

6. Fallblatt-System

Fallblatt-System ist eine Funktion, um Fallblätter am Boden hinzuzufügen. Nachdem ein Blatt gezeichnet wurde, gibt es eine Wahrscheinlichkeit, ein zusätzliches Fallblatt mit derselben Form im Bodenbereich zu zeichnen.

Es verbessert die Atmosphäre des Baumes.

6.1 Fallblattdichte

Fallblattdichte ist ein Parameter, der die Wahrscheinlichkeit bestimmt, dass Blätter zu Boden-Fallblättern konvertiert werden. Der Wertbereich ist 0,0 ~ 1,0. Je größer der Wert, desto mehr Fallblätter.

Die tatsächliche Anzahl der Fallblätter hängt auch von der Dichte der Baumkrone (der Anzahl der Blätter am Baum) ab.

6.2 Fallblattstreuung

Fallblattstreuung ist ein Parameter, der den Verteilungsbereich der Fallblätter bestimmt, speziell den horizontalen Versatzbereich (in Pixeln) relativ zum ursprünglichen Blatt und den vertikalen Versatzbereich (in Pixeln) relativ zur Bodenposition. Je größer der Wert, desto größer der Verteilungsbereich der Fallblätter.

Der vertikale Versatz relativ zur Bodenposition verwendet eine Dreiecksverteilung (Durchschnitt von zwei gleichmäßig zufälligen Werten).

Der horizontale Versatz relativ zum ursprünglichen Blatt verwendet eine Gleichverteilung, da die Zufälligkeit einer großen Anzahl von Blättern bereits einen ähnlichen Effekt wie eine Normalverteilung erzielt.

7. Parametergruppen-System

Parametergruppe ist das zentrale Abstraktionskonzept von Fraktalbaum Desktop-Hintergrundbild. Eine einzelne Baumart-Konfiguration kann eine oder mehrere Parametergruppen enthalten, jede Gruppe mit einem unabhängigen Parametersatz.

7.0 Warum Parametergruppen benötigt werden

Stellen Sie sich vor, Sie möchten einen Baum mit einem geraden Stamm, aber einer ausgebreiteten Krone entwerfen.

Wenn Sie nur einen Parametersatz verwenden, können Stamm und Krone nur dieselbe Form haben, was komplexe Effekte unmöglich macht.

Tatsächlich war dies das reale Problem, das ich beim Entwurf von Ahorn angetroffen habe.

Dafür hatte ich zwei Ideen:

Die erste Idee war, die y-Koordinate des Astes in "Parameter von Parametern" einzubeziehen, was direkt Parameter ermöglichen könnte, die sich mit der Höhe ändern. Mit anderen Worten, Parameter könnten als Funktionen von y geschrieben werden.

Dies entspricht gut dem Denken eines Programmierers und kann eine komplexere und präzisere Formkontrolle erreichen. Es ist jedoch offensichtlich wenig intuitiv. In der Praxis erfordert es ziemlich komplexes abstraktes Denken und den Einsatz von Taschenrechnern oder sogar Werkzeugen wie Desmos, um Funktionsgraphen zu untersuchen und Parameter wiederholt anzupassen.

Daher wählte ich die zweite Idee: das Parametergruppen-System.

Durch direkte Definition mehrerer Parametersätze und Anwendung verschiedener Parametergruppen auf verschiedenen Höhen können die Formen von Stamm und Krone unterschieden werden.

Bei der tatsächlichen Entwicklung entschied ich mich für eine inverse gewichtete Mischung von Parametern aus verschiedenen Parametergruppen für sanfte Übergänge.

7.1 Grundlegende Operationen

In Mehrere Parametergruppen-Steuerung:

  • Dropdown: Wählen Sie eine Parametergruppe zur Bearbeitung. Klicken Sie, um den Parametergruppennamen zu ändern.
  • 「+」: Fügen Sie eine neue Parametergruppe hinzu (kopiert die Parameter der aktuell ausgewählten Parametergruppe).
  • 「✕」: Löschen Sie die aktuell ausgewählte Parametergruppe.

7.2 Typische effektive Höhe

Typische effektive Höhe ist die Ankerhöhe (in Pixeln) dieser Parametergruppe, die für die gewichtete Mischungsberechnung mehrerer Parametergruppen verwendet wird.

Sie steuert die ungefähre Position, an der die Parametergruppe wirksam wird. Je näher der Ast an der typischen effektiven Höhe vom Boden ist, desto höher ist das Gewicht der Parametergruppe.

Einfach ausgedrückt werden Parameter in der Nähe der Typischen effektiven Höhe wirksam.

7.3 Gewichtungsberechnung

Wenn sich ein Ast auf der Höhe $h$ befindet, ist das Ursprungsgewicht der Gruppe $i$:

w_i = \min\left(\frac{1}{|h - t_i|},\ 128\right)

Wobei $t_i$ die typische Höhe der Gruppe $i$ ist. Das Gewicht wird auf 128 festgelegt, wenn der Abstand 0 ist, um eine Division durch Null zu vermeiden. Nach der Normalisierung aller Gewichte:

  • Gewöhnliche Skalarparameter (Winkelbereich, Abnahmebereich, minimaler Ast, maximale Tiefe, Blattschwellenwert, Fallblattparameter, Stammlänge, Astneigung): Gewichteter Durchschnitt.
  • Stamm/Blatt-Einstellungen: Direkte Verwendung der Farbliste der Gruppe mit dem größten Gewicht.
  • Verteilungskurven (Winkelverteilung, Abnahmeverteilung): Gewichteter Durchschnitt pro Balken.

Wenn es nur eine Parametergruppe gibt, wird die Mischungsberechnung übersprungen und alle Parameter dieser Gruppe direkt verwendet.

7.4 Referenzbeispiel

Der Standardbaum Ahorn hat zwei Parametergruppen: "Stamm" und "Krone". Der einzige Unterschied zwischen beiden ist der Verzweigungswinkelbereich, der Längenabnahmebereich und die Typische effektive Höhe.

Parametergruppe "Stamm":

  • Die typische effektive Höhe ist 0, was bedeutet, dass je näher am Boden, desto höher das Gewicht der Parametergruppe.
  • Der Verzweigungswinkelbereich ist 0 ~ 10 Gleichverteilung, was die aufrechte Form des Stammes beibehält.
  • Der Längenabnahmebereich ist 0,8 ~ 0,95 Gleichverteilung, was sicherstellt, dass der Stamm lang ist und die unteren Äste reduziert werden.

Parametergruppe "Krone":

  • Die typische effektive Höhe ist 180, was grundsätzlich bedeutet, dass je weiter vom Boden entfernt, desto höher das Gewicht der Parametergruppe.
  • Der Verzweigungswinkelbereich ist 0 ~ 60 mit hoher Wahrscheinlichkeit an beiden Enden, was die flauschige Form der Krone beibehält.
  • Der Längenabnahmebereich ist 0,3 ~ 0,95 mit hoher Wahrscheinlichkeit am großen Ende und niedriger, aber möglicher Wahrscheinlichkeit am kleinen Ende, was die geschichtete Form der Krone beibehält und die Vielfalt der Krone erhöht.

Baumarten importieren/exportieren

Das Programm speichert alle benutzerdefinierten Baumarten in .json-Dateien im Ordner AppData\Local\Fraktalbaum Desktop-Hintergrundbild\trees. Sie können auf die Schaltfläche Baumordner öffnen unten links im Parametereinstellungsfenster klicken, um ihn schnell zu öffnen.

  • Um eine Baumart zu importieren, platzieren Sie einfach die zu importierende .json-Datei in diesem Ordner und starten Sie das Programm neu.
  • Um eine Baumart zu exportieren, kopieren Sie einfach die zu exportierende .json-Datei.

Rendering-Fenster

Klicken Sie auf eine beliebige Baumart-Schaltfläche in der Hauptschnittstelle oder auf Vorschau in der Bearbeitungsschnittstelle, um das Fraktalbaum-Rendering-Fenster zu öffnen. Ziehen und Zoomen werden unterstützt.

  • Aktualisieren: Setzt den zufälligen Startwert zurück und zeichnet neu. Jeder Klick erzeugt eine andere Form.
  • Als Bild speichern: Rendert als PNG in Fenstergröße, speichert auf dem Desktop als _.png. Das Fenster bleibt nach Abschluss geöffnet.
  • Als Desktop-Hintergrundbild festlegen: Rendert als BMP bei 2x Bildschirmauflösung, ruft die System-API auf, um als Windows-Desktop-Hintergrundbild festzulegen. Das Fenster schließt automatisch nach Abschluss.

Arbre Fractal Fond d'Écran - Guide d'Utilisation

Introduction

Arbre Fractal Fond d'Écran peut générer aléatoirement un bel arbre. Grâce à son puissant système de paramètres, il peut simuler presque tous les types d'arbres dans le monde, y compris ceux qui existent dans la nature et ceux qui n'existent que dans votre imagination.

Qu'est-ce qu'un Arbre Fractal?

Un Arbre Fractal est une structure d'arbre générée récursivement — chaque branche a plusieurs sous-branches, chaque sous-branch contient plusieurs sous-branches, et ainsi de suite, formant une structure arborescente.

Dans Arbre Fractal Fond d'Écran, pour réduire les calculs de rendu et améliorer la contrôlabilité, nous n'utilisons que des arbres binaires, ce qui signifie que chaque branche a au maximum deux sous-branches.

Pour obtenir une structure d'arbre aussi réaliste que possible, Arbre Fractal Fond d'Écran fournit de nombreux paramètres pour contrôler la distance et l'angle entre les nœuds, la couleur des lignes, etc.

Introduction aux Paramètres

Si vous êtes nouveau avec Arbre Fractal Fond d'Écran, ne vous laissez pas intimider par le grand nombre de paramètres — vous n'avez besoin d'ajuster que quelques paramètres clés pour créer un arbre unique. Essayez de modifier la Plage d'Angle de Bifurcation et la Plage de Décroissance de Longueur pour voir comment la forme de l'arbre change ; ajustez ensuite la couleur dans les Paramètres des Feuilles pour ajouter de la couleur à votre arbre. Une fois familiarisé avec les opérations de base, vous pouvez explorer davantage les Groupes de Paramètres et les Courbes de Distribution pour faire de chaque arbre une véritable œuvre d'art.

Voici un tutoriel de base sur les paramètres ↓

1. Plage d'Angle de Bifurcation

Plage d'Angle de Bifurcation contrôle la plage de l'angle de déviation (en degrés) des sous-branches par rapport à leur branche parent. La courbe de distribution contrôle la probabilité des valeurs d'angle dans la plage (bien sûr, les courbes de distribution ne sont pas couvertes dans ce tutoriel de base, gardez donc la distribution uniforme par défaut).

  • Quand la Plage d'Angle de Bifurcation est globalement grande, la cime de l'arbre apparaît clairsemée et duveteuse.
  • Quand la Plage d'Angle de Bifurcation est globalement petite, la cime de l'arbre apparaît dense et compacte.
  • Quand la Plage d'Angle de Bifurcation a un intervalle large, la cime de l'arbre apparaît chaotique et désordonnée.
  • Quand la Plage d'Angle de Bifurcation a un intervalle étroit, la cime de l'arbre apparaît ordonnée et régulière.

2. Plage de Décroissance de Longueur

Plage de Décroissance de Longueur contrôle la plage du rapport de la longueur de la sous-branch par rapport à la branche parent. La courbe de distribution contrôle la probabilité des valeurs de rapport dans la plage (là encore, les courbes de distribution ne sont pas couvertes dans ce tutoriel de base, gardez la distribution uniforme par défaut).

Par exemple, si une bifurcation obtient une valeur aléatoire de 0,7 dans la plage prédéfinie, la longueur de la sous-branch sera de 70% de la longueur de la branche parent.

  • Quand la Plage de Décroissance de Longueur est globalement grande, la cime de l'arbre apparaît énorme.
  • Quand la Plage de Décroissance de Longueur est globalement petite, la cime de l'arbre apparaît petite et clairsemée.
  • Quand la Plage de Décroissance de Longueur a un intervalle large, la cime de l'arbre apparaît variée et étagée.
  • Quand la Plage de Décroissance de Longueur a un intervalle étroit, la cime de l'arbre apparaît uniforme et dense.

Notez que sans exigences particulières, la valeur maximale de la Plage de Décroissance de Longueur ne devrait pas dépasser 0,95, sinon la cime de l'arbre sera trop dense, ou même croîtra indéfiniment (limitée uniquement par la Profondeur Maximale), ce qui affecte non seulement les performances de rendu mais rend aussi la cime extrêmement artificielle.

3. Paramètres des Feuilles

Paramètres des Feuilles contrôle l'apparence des feuilles. En ajustant la couleur, le poids, la densité et la largeur de ligne des feuilles, vous pouvez obtenir différents effets de feuilles. (Pour le tutoriel de base, modifier uniquement les couleurs suffit.)

Vous pouvez commencer par ajouter plusieurs nuances de vert avec une certaine variation pour voir l'effet.

4. Couleur de Fond

Comme son nom l'indique, Couleur de Fond contrôle la couleur de fond du canevas.

Vous pouvez utiliser un ton clair similaire aux feuilles pour un aspect plus naturel.

Pour des suggestions de couleurs plus professionnelles, vous pouvez consulter l'Outil de Cercle de Couleurs | Adobe Express.

Paramètres Avancés

Cette section fournit des définitions détaillées et l'utilisation de chaque paramètre.

Tous les paramètres en pixels représentent des longueurs lorsque la hauteur de la fenêtre est de 1080 pixels standard. Le rendu réel est mis à l'échelle proportionnellement en fonction de la hauteur de la fenêtre.

1. Paramètres Globaux

Les paramètres globaux ne sont inclus dans aucun groupe de paramètres et affectent directement le résultat.

1.1 Nom de l'Arbre

Nom de l'Arbre est utilisé pour donner un nom à votre arbre.

Il est affiché dans la liste de l'interface principale et utilisé comme préfixe du nom de fichier par défaut pour les images exportées. Les chaînes vides sont automatiquement corrigées en "Sans titre".

1.2 Couleur de Fond

Couleur de Fond est utilisée pour contrôler la couleur de fond du canevas.

Utilisée pour remplir la couleur de fond de la fenêtre et la couleur de fond de l'image de fond d'écran.

2. Paramètres de Forme de Base

Les paramètres suivants contrôlent directement la forme de base et les conditions d'arrêt de l'arbre.

2.1 Longueur du Tronc

Longueur du Tronc est la longueur initiale (en pixels) de la première branche émanant de la racine de l'arbre. La racine est positionnée à 10% au-dessus du bas du canevas.

Elle détermine l'échelle de base de tout l'arbre et affecte directement la taille finale de l'arbre.

2.2 Longueur Minimale de Branche

Longueur Minimale de Branche est l'une des conditions d'arrêt de l'arbre. Quand la longueur d'une branche tombe en dessous de ce seuil, aucune sous-branch supplémentaire n'est générée.

Elle contrôle la finesse des extrémités de la cime de l'arbre et la taille de la cime.

2.3 Profondeur Maximale

Profondeur Maximale est l'une des conditions d'arrêt de l'arbre. Quand le niveau de récursion d'une branche (la racine est le niveau 0, les sous-branches de la racine sont le niveau 1, etc.) atteint cette valeur, la bifurcation s'arrête.

Elle empêche la cime de l'arbre de devenir trop dense ou de croître indéfiniment, ce qui affecterait les performances ou ferait même geler l'application.

Notez que dans les premiers stades de création, il est recommandé de régler la Profondeur Maximale sur 10 ~ 30, puis d'ajuster selon les besoins pour éviter les gels lors des tests dus à des paramètres inappropriés.

2.4 Seuil de Feuille

Seuil de Feuille détermine si la branche actuelle doit être rendue comme une brindille ou comme une feuille. Basé sur la longueur de la branche pour déterminer le rôle visuel. Quand la longueur de la branche actuelle est inférieure à ce seuil, la branche est rendue comme une feuille ; au-dessus du seuil, elle est rendue comme un tronc.

Il contrôle l'effet visuel des feuilles et affecte la forme de la cime de l'arbre.

Il n'est généralement pas recommandé de l'utiliser pour ajuster directement la densité des feuilles, car cela est lié à la forme des feuilles. D'autres paramètres doivent être ajustés à la place.

2.5 Inclinaison Maximale de Branche

Inclinaison Maximale de Branche est l'une des conditions d'arrêt de l'arbre. Elle contrôle la limite supérieure (en degrés) de l'angle absolu des sous-branches (l'angle par rapport à la direction verticale vers le haut). Toute sous-branch dont l'angle dépasse cette valeur ne sera pas créée et se terminera directement.

Elle empêche les branches de pendre et maintient la forme de l'arbre.

3. Plage d'Angle de Bifurcation

Plage d'Angle de Bifurcation est la plage d'intervalle de l'angle de déviation échantillonné indépendamment pour chaque sous-branch. Deux sous-branches dévient chacune vers un côté de la branche parent. La plage d'échantillonnage est [Minimum, Maximum] (degrés).

Elle contrôle la forme des branches et affecte l'ordre et le schéma de croissance des branches.

3.1 Courbe de Distribution de Probabilité d'Angle

Ouvrez l'éditeur de graphique à barres via le bouton Courbe de Distribution : cliquez sur les barres pour ajuster les hauteurs des barres, avec une plage de hauteur de 0,02 ~ 1,0.

Méthode d'échantillonnage pour chaque sous-branch: Divisez l'intervalle d'angle en 8 sous-intervalles égaux, sélectionnez d'abord un sous-intervalle en utilisant les hauteurs des barres de la courbe de distribution comme poids, puis prenez une valeur aléatoire uniforme dans ce sous-intervalle. Si aucune courbe de distribution n'est spécifiée, une valeur aléatoire uniforme est prise dans l'intervalle global.

En résumé, plus la barre est haute, plus la probabilité que les valeurs proches soient sélectionnées comme angles de branches est élevée.

3.2 Exemple de Référence

L'arbre par défaut Pin a une plage d'angle de bifurcation de [0, 90], avec une courbe de distribution de probabilité d'angle approximativement comme suit :

0°
12.86°
25.71°
38.57°
51.43°
64.29°
77.14°
90°

On voit que les angles de bifurcation du pin sont plus susceptibles d'être autour de 0 et 90 degrés. C'est-à-dire que les branches du pin sont plus susceptibles d'être presque parallèles ou perpendiculaires à la branche parent, plutôt que de croître en diagonale.

L'effet réel est que les pins croissent surtout droit, mais font parfois de grands changements de direction, fournissant une forme similaire au pin tordu et vigoureux réel.

Être plus susceptible d'être parallèle à la branche parent permet de maintenir la forme de base de la cime sans être trop chaotique.

4. Plage de Décroissance de Longueur

Plage de Décroissance de Longueur est la plage d'intervalle du rapport de décroissance de longueur échantillonné indépendamment pour chaque sous-branch. La longueur réelle de la sous-branch = longueur de la branche parent × rapport de décroissance échantillonné. La plage d'échantillonnage est [Minimum, Maximum] (rapport de décroissance).

4.1 Courbe de Distribution de Probabilité de Décroissance

Similaire à 3.1 Courbe de Distribution de Probabilité d'Angle.

Ouvrez l'éditeur de graphique à barres via le bouton Courbe de Distribution : cliquez sur les barres pour ajuster les hauteurs des barres, avec une plage de hauteur de 0,02 ~ 1,0.

Méthode d'échantillonnage pour chaque sous-branch: Divisez l'intervalle d'angle en 8 sous-intervalles égaux, sélectionnez d'abord un sous-intervalle en utilisant les hauteurs des barres de la courbe de distribution comme poids, puis prenez une valeur aléatoire uniforme dans ce sous-intervalle. Si aucune courbe de distribution n'est spécifiée, une valeur aléatoire uniforme est prise dans l'intervalle global.

En résumé, plus la barre est haute, plus la probabilité que les valeurs proches soient sélectionnées comme rapports de décroissance est élevée.

4.2 Exemple de Référence

L'arbre par défaut Arbre Vert a une plage de décroissance de longueur de [0,6, 0,94], avec une courbe de distribution de probabilité de décroissance approximativement comme suit :

0.60
0.65
0.70
0.75
0.79
0.84
0.89
0.94

On voit que les rapports de décroissance de l'arbre vert sont plus susceptibles d'être autour de 0,60 et 0,94. C'est-à-dire que les branches de l'arbre vert sont soit presque de la même longueur que la branche parent, soit nettement plus courtes.

L'effet réel est que l'arbre vert a parfois des branches étendues et parfois des feuilles rassemblées, fournissant une forme étagée et variée.

Il est à noter que si l'arbre vert choisit une branche courte trop tôt, cela réduira directement la hauteur de tout l'arbre, entraînant une plus grande variation de la hauteur de l'arbre.

5. Paramètres de Tronc/Feuilles

Paramètres de Tronc/Feuilles est une collection de paramètres qui déterminent l'effet visuel des troncs et des feuilles, sans affecter la structure de l'arbre.

Les troncs et les feuilles ont chacun des listes de couleurs indépendantes. Chaque entrée de couleur contient trois attributs :

5.1 Couleur

Couleur est un paramètre qui détermine la couleur utilisée pour rendre les troncs ou les feuilles. Supporte la définition de plusieurs couleurs pour les troncs/feuilles, sélectionnées aléatoirement.

5.2 Poids

Poids est un paramètre qui détermine la probabilité que cette couleur soit sélectionnée lors du dessin d'un nouveau tronc ou feuille. Plus la valeur est grande, plus la proportion de cette couleur dans les troncs/feuilles est élevée.

5.3 Rapport de Largeur de Ligne

Rapport de Largeur de Ligne est un paramètre qui détermine le coefficient de largeur des troncs ou des feuilles. Plus la valeur est grande, plus le tronc/la feuille est épais. L'épaisseur réelle dépend aussi de la longueur de la branche.

Comportement avec Plusieurs Groupes de Paramètres: Les paramètres de tronc/feuilles ne participent pas à la moyenne pondérée numérique de plusieurs groupes de paramètres. Quand plusieurs groupes de paramètres existent, la liste de couleurs du groupe avec le plus grand poids est utilisée directement.

6. Système de Feuilles Mortes

Système de Feuilles Mortes est une fonction pour ajouter des feuilles mortes au sol. Après qu'une feuille soit dessinée, il y a une probabilité de dessiner une feuille morte supplémentaire de même forme dans la zone du sol.

Il améliore l'ambiance de l'arbre.

6.1 Densité de Feuilles Mortes

Densité de Feuilles Mortes est un paramètre qui détermine la probabilité que les feuilles soient converties en feuilles mortes au sol. La plage de valeurs est 0,0 ~ 1,0. Plus la valeur est grande, plus il y a de feuilles mortes.

Le nombre réel de feuilles mortes dépend aussi de la densité de la cime de l'arbre (le nombre de feuilles sur l'arbre).

6.2 Dispersion des Feuilles Mortes

Dispersion des Feuilles Mortes est un paramètre qui détermine la plage de distribution des feuilles mortes, spécifiquement la plage de décalage horizontal (en pixels) par rapport à la feuille originale et la plage de décalage vertical (en pixels) par rapport à la position du sol. Plus la valeur est grande, plus la plage de distribution des feuilles mortes est large.

Le décalage vertical par rapport à la position du sol utilise une distribution triangulaire (moyenne de deux valeurs aléatoires uniformes).

Le décalage horizontal par rapport à la feuille originale utilise une distribution uniforme, car l'aléatoire d'un grand nombre de feuilles atteint déjà un effet similaire à une distribution normale.

7. Système de Groupes de Paramètres

Groupe de Paramètres est le concept d'abstraction central d'Arbre Fractal Fond d'Écran. Une configuration d'espèce d'arbre unique peut contenir un ou plusieurs groupes de paramètres, chaque groupe ayant un ensemble indépendant de paramètres.

7.0 Pourquoi des Groupes de Paramètres sont Nécessaires

Imaginez que vous voulez concevoir un arbre avec un tronc droit mais une cime étendue.

Si vous n'utilisez qu'un seul ensemble de paramètres, le tronc et la cime ne peuvent avoir la même forme, rendant les effets complexes impossibles.

En fait, c'était le problème réel que j'ai rencontré en concevant l'Érable.

Pour cela, j'ai eu deux idées :

La première idée était d'incorporer la coordonnée y de la branche dans les "paramètres de paramètres", ce qui pouvait implémenter directement des paramètres qui changent avec la hauteur. En d'autres termes, les paramètres pouvaient être écrits comme des fonctions de y.

Cela correspond bien à la pensée d'un programmeur et peut réaliser un contrôle de forme plus complexe et précis. Cependant, c'est clairement peu intuitif. En pratique, cela nécessite une pensée abstraite assez complexe et l'utilisation de calculatrices ou même d'outils comme Desmos pour étudier les graphiques de fonctions et ajuster les paramètres de manière répétée.

J'ai donc choisi la deuxième idée : le système de groupes de paramètres.

En définissant directement plusieurs ensembles de paramètres et en appliquant différents groupes de paramètres à différentes hauteurs, les formes du tronc et de la cime peuvent être distinguées.

Pendant le développement réel, pour des transitions fluides, j'ai choisi d'appliquer un mélange pondéré inverse des paramètres de différents groupes de paramètres.

7.1 Opérations de Base

Dans Contrôle de Plusieurs Groupes de Paramètres:

  • Liste Déroulante: Sélectionnez un groupe de paramètres à éditer. Cliquez pour modifier le nom du groupe de paramètres.
  • 「+」: Ajoutez un nouveau groupe de paramètres (copie les paramètres du groupe de paramètres actuellement sélectionné).
  • 「✕」: Supprimez le groupe de paramètres actuellement sélectionné.

7.2 Hauteur Efficace Typique

Hauteur Efficace Typique est la hauteur d'ancrage (en pixels) de ce groupe de paramètres, utilisée pour les calculs de mélange pondéré de plusieurs groupes de paramètres.

Elle contrôle la position approximative où le groupe de paramètres prend effet. Plus la branche est proche de la hauteur efficace typique du sol, plus le poids du groupe de paramètres est élevé.

En résumé, les paramètres prendront effet près de la Hauteur Efficace Typique.

7.3 Calcul du Poids

Quand une branche est à la hauteur $h$, le poids d'origine du groupe $i$ est :

w_i = \min\left(\frac{1}{|h - t_i|},\ 128\right)

Où $t_i$ est la hauteur typique du groupe $i$. Le poids est fixé à 128 quand la distance est 0 pour éviter la division par zéro. Après normalisation de tous les poids :

  • Paramètres scalaires ordinaires (plage d'angle, plage de décroissance, branche minimale, profondeur maximale, seuil de feuille, paramètres de feuilles mortes, longueur du tronc, inclinaison de branche): Moyenne pondérée.
  • Paramètres de Tronc/Feuilles: Utilisation directe de la liste de couleurs du groupe au poids le plus élevé.
  • Courbes de Distribution (distribution d'angle, distribution de décroissance): Moyenne pondérée par barre.

Quand il n'y a qu'un seul groupe de paramètres, le calcul de mélange est ignoré et tous les paramètres de ce groupe sont utilisés directement.

7.4 Exemple de Référence

L'arbre par défaut Érable a deux groupes de paramètres : "Tronc" et "Cime". La seule différence entre eux est la Plage d'Angle de Bifurcation, la Plage de Décroissance de Longueur et la Hauteur Efficace Typique.

Groupe de paramètres "Tronc":

  • La hauteur efficace typique est 0, ce qui signifie que plus près du sol, le poids du groupe de paramètres est élevé.
  • La plage d'angle de bifurcation est 0 ~ 10 distribution uniforme, maintenant la forme droite du tronc.
  • La plage de décroissance de longueur est 0,8 ~ 0,95 distribution uniforme, assurant que le tronc est long et réduisant les branches inférieures.

Groupe de paramètres "Cime":

  • La hauteur efficace typique est 180, ce qui signifie fondamentalement que plus loin du sol, le poids du groupe de paramètres est élevé.
  • La plage d'angle de bifurcation est 0 ~ 60 avec une probabilité élevée aux deux extrémités, maintenant la forme duveteuse de la cime.
  • La plage de décroissance de longueur est 0,3 ~ 0,95 avec une probabilité élevée à l'extrémité grande et une probabilité basse mais possible à l'extrémité petite, maintenant la forme étagée de la cime et augmentant la diversité de la cime.

Importer/Exporter des Espèces d'Arbres

  • Pour exporter une espèce d'arbre, copiez simplement le fichier .json à exporter.

Введение

Фрактальное Дерево Обои для Рабочего Стола может случайным образом генерировать красивое дерево. Благодаря мощной системе параметров, оно может моделировать почти любой тип дерева в мире, включая те, которые существуют в природе, и те, которые существуют только в вашем воображении.

Что такое Фрактальное Дерево?

Фрактальное Дерево — это рекурсивно генерируемая древовидная структура — каждая ветвь имеет несколько подветвей, каждая подветвь содержит несколько подветвей, и так далее, образуя древовидную структуру.

В Фрактальное Дерево Обои для Рабочего Стола, чтобы уменьшить вычисления при рендеринге и улучшить управляемость, мы используем только двоичные деревья, что означает, что каждая ветвь имеет максимум две подветви.

Чтобы получить максимально реалистичную структуру дерева, Фрактальное Дерево Обои для Рабочего Стола предоставляет множество параметров для управления расстоянием и углом между узлами, цветом линий и т.д.

Введение в Параметры

Если вы новичок в Фрактальное Дерево Обои для Рабочего Стола, не пугайтесь большого количества параметров — вам нужно настроить лишь несколько ключевых параметров, чтобы создать уникальное дерево. Попробуйте изменить Диапазон Угла Ветвления и Диапазон Убывания Длины, чтобы увидеть, как меняется форма дерева; затем настройте цвет в Настройках Листьев, чтобы добавить цвет вашему дереву. Когда вы освоитесь с базовыми операциями, вы можете дальше изучать Группы Параметров и Кривые Распределения, чтобы сделать каждое дерево настоящим произведением искусства.

1. Диапазон Угла Ветвления

Диапазон Угла Ветвления управляет диапазоном угла отклонения (в градусах) подветвей относительно родительской ветви. Кривая распределения управляет вероятностью значений угла в диапазоне (конечно, кривые распределения выходят за рамки этого базового учебника, так что оставьте равномерное распределение по умолчанию).

  • Когда Диапазон Угла Ветвления в целом велик, крона дерева выглядит разреженной и пушистой.
  • Когда Диапазон Угла Ветвления в целом мал, крона дерева выглядит плотной и густой.
  • Когда Диапазон Угла Ветвления имеет широкий интервал, крона дерева выглядит хаотичной и беспорядочной.
  • Когда Диапазон Угла Ветвления имеет узкий интервал, крона дерева выглядит упорядоченной и ровной.

2. Диапазон Убывания Длины

Диапазон Убывания Длины управляет диапазоном отношения длины подветви к длине родительской ветви. Кривая распределения управляет вероятностью значений отношения в диапазоне (опять же, кривые распределения выходят за рамки этого базового учебника, так что оставьте равномерное распределение по умолчанию).

Например, если при ветвлении случайным образом выбрано значение 0,7 из предустановленного диапазона, длина подветви составит 70% от длины родительской ветви.

  • Когда Диапазон Убывания Длины в целом велик, крона дерева выглядит огромной.
  • Когда Диапазон Убывания Длины в целом мал, крона дерева выглядит маленькой и разреженной.
  • Когда Диапазон Убывания Длины имеет широкий интервал, крона дерева выглядит разнообразной и многослойной.
  • Когда Диапазон Убывания Длины имеет узкий интервал, крона дерева выглядит равномерной и плотной.

Обратите внимание, что без особых требований максимальное значение Диапазона Убывания Длины не должно превышать 0,95, иначе крона дерева будет слишком плотной или будет расти бесконечно (ограничено только Максимальной Глубиной), что не только влияет на производительность рендеринга, но и делает крону крайне неестественной.

3. Настройки Листьев

Настройки Листьев управляют внешним видом листьев. Настраивая цвет, вес, плотность и толщину линии листьев, вы можете достичь различных эффектов листьев. (Для базового учебника достаточно изменить только цвета.)

Вы можете начать с добавления нескольких оттенков зелёного с некоторой вариацией, чтобы увидеть эффект.

4. Цвет Фона

Как следует из названия, Цвет Фона управляет цветом фона холста.

Вы можете использовать светлый оттенок цвета листьев для более естественного вида.

Для более профессиональных рекомендаций по цвету вы можете обратиться к Инструмент Цветового Круга | Adobe Express.

Продвинутые Параметры

Этот раздел предоставляет подробные определения и использование каждого параметра.

Все параметры в пикселях представляют длины при стандартной высоте окна 1080 пикселей. Фактический рендер масштабируется пропорционально в зависимости от высоты окна.

1. Глобальные Настройки

Глобальные настройки не включены ни в одну группу параметров и напрямую влияют на результат.

1.1 Имя Дерева

Имя Дерева используется для названия вашего дерева.

Отображается в списке главного интерфейса и используется как префикс имени файла по умолчанию для экспортируемых изображений. Пустые строки автоматически исправляются на "Безымянный".

1.2 Цвет Фона

Цвет Фона используется для управления цветом фона холста.

Используется для заполнения цвета фона окна и цвета фона изображения обоев.

2. Параметры Основной Формы

Следующие параметры напрямую управляют основной формой и условиями завершения дерева.

2.1 Длина Ствола

Длина Ствола — это начальная длина (в пикселях) первой ветви, исходящей от корня дерева. Корень дерева расположен на 10% выше нижней части холста.

Определяет базовый масштаб всего дерева, напрямую влияя на окончательный размер дерева.

2.2 Минимальная Длина Ветви

Минимальная Длина Ветви — одно из условий завершения дерева. Когда длина ветви падает ниже этого порога, подветви больше не генерируются.

Управляет детализацией концов кроны дерева и размером кроны.

2.3 Максимальная Глубина

Максимальная Глубина — одно из условий завершения дерева. Когда уровень рекурсии ветви (корень — уровень 0, подветви от корня — уровень 1 и т.д.) достигает этого значения, ветвление прекращается.

Предотвращает слишком плотную крону или бесконечный рост, что влияет на производительность или даже вызывает зависание приложения.

Обратите внимание, что на ранних этапах создания рекомендуется установить Максимальную Глубину на 10 ~ 30, а затем корректировать по необходимости, чтобы избежать зависаний при тестировании из-за неподходящих параметров.

2.4 Порог Листа

Порог Листа определяет, должна ли текущая ветвь отображаться как веточка или как лист. Основан на длине ветви для определения визуальной роли. Когда длина текущей ветви ниже этого порога, ветвь отображается как лист; выше порога — как ствол.

Управляет визуальным эффектом листьев, влияя на форму кроны дерева.

Обычно не рекомендуется использовать это для прямой настройки плотности листьев, так как это связано с формой листьев. Следует настраивать другие параметры.

2.5 Максимальный Наклон Ветви

Максимальный Наклон Ветви — одно из условий завершения дерева. Управляет верхним пределом (в градусах) абсолютного угла подветвей (угол относительно вертикального направления вверх). Любая подветвь, угол которой превышает это значение, не будет создана и завершится напрямую.

Предотвращает провисание ветвей и сохраняет форму дерева.

3. Диапазон Угла Ветвления

Диапазон Угла Ветвления — это диапазон интервала угла отклонения, независимо выборки для каждой подветви. Две подветви отклоняются в каждую сторону от родительской ветви соответственно. Диапазон выборки — [Минимум, Максимум] (градусы).

Управляет формой ветвей, влияя на упорядоченность и схему роста ветвей.

3.1 Кривая Распределения Вероятности Угла

Откройте редактор столбчатой диаграммы через кнопку Кривая Распределения: кликайте по столбцам для настройки высоты столбцов, диапазон высоты 0,02 ~ 1,0.

Метод выборки для каждой подветви: Разделите интервал угла на 8 равных подинтервалов, сначала выберите подинтервал, используя высоты столбцов кривой распределения как веса, затем возьмите равномерно случайное значение в этом подинтервале. Если кривая распределения не указана, берётся равномерно случайное значение из глобального интервала.

Проще говоря, чем выше столбец, тем выше вероятность, что близкие значения будут выбраны как углы ветвей.

Дерево по умолчанию Сосна имеет диапазон угла ветвления [0, 90], с кривой распределения вероятности угла примерно следующей:

Видно, что углы ветвления сосны с большей вероятностью будут около 0 и 90 градусов. То есть ветви сосны скорее будут почти параллельны или перпендикулярны родительской ветви, чем будут расти по диагонали.

Фактический эффект в том, что сосны в основном растут прямо, но иногда резко меняют направление, обеспечивая форму, похожую на изогнутый и мощный вид настоящих сосен.

Большая вероятность быть параллельными родительской ветви помогает сохранить базовую форму кроны дерева, не становясь слишком хаотичной.

4. Диапазон Убывания Длины

Диапазон Убывания Длины — это диапазон интервала коэффициента убывания длины, независимо выборки для каждой подветви. Фактическая длина подветви = длина родительской ветви × выбранный коэффициент убывания. Диапазон выборки — [Минимум, Максимум] (коэффициент убывания).

4.1 Кривая Распределения Вероятности Убывания

Аналогично 3.1 Кривая Распределения Вероятности Угла.

Откройте редактор столбчатой диаграммы через кнопку Кривая Распределения: кликайте по столбцам для настройки высоты столбцов, диапазон высоты 0,02 ~ 1,0.

Метод выборки для каждой подветви: Разделите интервал угла на 8 равных подинтервалов, сначала выберите подинтервал, используя высоты столбцов кривой распределения как веса, затем возьмите равномерно случайное значение в этом подинтервале. Если кривая распределения не указана, берётся равномерно случайное значение из глобального интервала.

Проще говоря, чем выше столбец, тем выше вероятность, что близкие значения будут выбраны как коэффициенты убывания.

4.2 Пример для Справки

Дерево по умолчанию Зелёное Дерево имеет диапазон убывания длины [0,6, 0,94], с кривой распределения вероятности убывания примерно следующей:

0.70

Видно, что коэффициенты убывания зелёного дерева с большей вероятностью будут около 0,60 и 0,94. То есть ветви зелёного дерева либо почти такой же длины, как родительская ветвь, либо заметно короче.

Фактический эффект в том, что зелёное дерево иногда имеет раскинутые ветви, а иногда собранные листья, обеспечивая многослойную и разнообразную форму.

Стоит отметить, что если зелёное дерево слишком рано выберет короткую ветвь, это напрямую уменьшит высоту всего дерева, что приведёт к большей вариации высоты дерева.

5. Настройки Ствола/Листьев

Настройки Ствола/Листьев — это набор параметров, определяющих визуальный эффект стволов и листьев, не затрагивая структуру дерева.

Стволы и листья имеют независимые списки цветов. Каждая запись цвета содержит три атрибута:

5.1 Цвет

Цвет — параметр, определяющий цвет, используемый для рендеринга стволов или листьев. Поддерживает установку нескольких цветов для стволов/листьев, выбираемых случайно.

5.2 Вес

Вес — параметр, определяющий вероятность выбора этого цвета при рисовании нового ствола или листа. Чем больше значение, тем выше доля этого цвета в стволах/листьях.

5.3 Коэффициент Толщины Линии

Коэффициент Толщины Линии — параметр, определяющий коэффициент толщины стволов или листьев. Чем больше значение, тем толще ствол/лист. Фактическая толщина также зависит от длины ветви.

Поведение при Нескольких Группах Параметров: Настройки ствола/листьев не участвуют в числовом взвешенном среднем нескольких групп параметров. Когда существует несколько групп параметров, напрямую используется список цветов группы с наибольшим весом.

6. Система Опавших Листьев

Система Опавших Листьев — функция для добавления опавших листьев на землю. После того как лист нарисован, есть вероятность нарисовать дополнительный опавший лист той же формы в области земли.

Улучшает атмосферу дерева.

6.1 Плотность Опавших Листьев

Плотность Опавших Листьев — параметр, определяющий вероятность преобразования листьев в опавшие листья на земле. Диапазон значений 0,0 ~ 1,0. Чем больше значение, тем больше опавших листьев.

Фактическое количество опавших листьев также зависит от плотности кроны дерева (количества листьев на дереве).

6.2 Рассеяние Опавших Листьев

Рассеяние Опавших Листьев — параметр, определяющий диапазон распределения опавших листьев, конкретно диапазон горизонтального смещения (в пикселях) относительно исходного листа и диапазон вертикального смещения (в пикселях) относительно положения земли. Чем больше значение, тем шире диапазон распределения опавших листьев.

Вертикальное смещение относительно положения земли использует треугольное распределение (среднее двух равномерных случайных значений).

Горизонтальное смещение относительно исходного листа использует равномерное распределение, так как случайность большого количества листьев уже достигает эффекта, аналогичного нормальному распределению.

7. Система Групп Параметров

Группа Параметров — это ключевая концепция абстракции Фрактальное Дерево Обои для Рабочего Стола. Конфигурация одного вида дерева может содержать одну или несколько групп параметров, каждая группа имеет независимый набор параметров.

7.0 Почему Нужны Группы Параметров

Представьте, что вы хотите спроектировать дерево с прямым стволом, но раскинутой кроной.

Если вы используете только один набор параметров, ствол и крона могут иметь только одинаковую форму, что делает сложные эффекты невозможными.

На самом деле, это реальная проблема, с которой я столкнулся при проектировании Клёна.

Для этого у меня было две идеи:

Первая идея — включить y-координату ветви в "параметры параметров", что могло бы напрямую реализовать параметры, меняющиеся с высотой. Другими словами, параметры могли бы быть записаны как функции от y.

Это хорошо соответствует мышлению программиста и может реализовать более сложный и точный контроль формы. Однако это явно неинтуитивно. На практике это требует довольно сложного абстрактного мышления и использования калькуляторов или даже инструментов типа Desmos для изучения графиков функций и повторной настройки параметров.

Поэтому я выбрал вторую идею — систему групп параметров.

Прямо определяя несколько наборов параметров и применяя разные группы параметров на разных высотах, можно различать формы ствола и кроны.

При фактической разработке для плавных переходов я выбрал обратное взвешенное смешивание параметров из разных групп параметров.

7.1 Базовые Операции

В Управление Несколькими Группами Параметров:

  • Выпадающий список: Выберите группу параметров для редактирования. Кликните для изменения имени группы параметров.
  • 「+」: Добавьте новую группу параметров (копирует параметры текущей выбранной группы).
  • 「✕」: Удалите текущую выбранную группу параметров.

7.2 Типичная Эффективная Высота

Типичная Эффективная Высота — это высота привязки (в пикселях) этой группы параметров, используемая для расчётов взвешенного смешивания нескольких групп параметров.

Управляет примерной позицией, где группа параметров вступает в силу. Чем ближе ветвь к типичной эффективной высоте от земли, тем выше вес группы параметров.

Проще говоря, параметры вступят в силу около Типичной Эффективной Высоты.

7.3 Расчёт Веса

Когда ветвь находится на высоте $h$, исходный вес группы $i$ равен:

w_i = \min\left(\frac{1}{|h - t_i|},\ 128\right)

Где $t_i$ — типичная высота группы $i$. Вес фиксируется на 128, когда расстояние равно 0, чтобы избежать деления на ноль. После нормализации всех весов:

  • Обычные скалярные параметры (диапазон угла, диапазон убывания, минимальная ветвь, максимальная глубина, порог листа, параметры опавших листьев, длина ствола, наклон ветви): Взвешенное среднее.
  • Настройки Ствола/Листьев: Прямое использование списка цветов группы с наибольшим весом.
  • Кривые Распределения (распределение угла, распределение убывания): Взвешенное среднее по столбцам.

Когда есть только одна группа параметров, расчёт смешивания пропускается, и все параметры этой группы используются напрямую.

7.4 Пример для Справки

Дерево по умолчанию Клён имеет две группы параметров: "Ствол" и "Крона". Единственное различие между ними — Диапазон Угла Ветвления, Диапазон Убывания Длины и Типичная Эффективная Высота.

Группа параметров "Ствол":

  • Типичная эффективная высота 0, что означает, что чем ближе к земле, тем выше вес группы параметров.
  • Диапазон угла ветвления 0 ~ 10 равномерное распределение, поддерживая прямую форму ствола.
  • Диапазон убывания длины 0,8 ~ 0,95 равномерное распределение, обеспечивая длинный ствол и уменьшая нижние ветви.

Группа параметров "Крона":

  • Типичная эффективная высота 180, что в основном означает, что чем дальше от земли, тем выше вес группы параметров.
  • Диапазон угла ветвления 0 ~ 60 с высокой вероятностью на обоих концах, поддерживая пушистую форму кроны.
  • Диапазон убывания длины 0,3 ~ 0,95 с высокой вероятностью на большом конце и низкой, но возможной вероятностью на малом конце, поддерживая многослойную форму кроны и увеличивая разнообразие кроны.

Окно Рендеринга

Кликните любую кнопку вида дерева в главном интерфейсе или Предпросмотр в интерфейсе редактирования, чтобы открыть окно рендеринга фрактального дерева. Поддерживаются перетаскивание и масштабирование.

  • Обновить: Сбрасывает случайное зерно и перерисовывает. Каждый клик производит разную форму.
  • Сохранить как Изображение: Рендерит в PNG в размере окна, сохраняет на рабочий стол как <Имя Дерева>_<Временная Метка>.png. Окно остаётся открытым после завершения.
  • Установить как Обои: Рендерит в BMP с 2x разрешением экрана, вызывает системный API для установки как обои рабочего стола Windows. Окно закрывается автоматически после завершения.
